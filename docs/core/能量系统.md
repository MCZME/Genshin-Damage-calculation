# Core - 能量系统 (Energy System)

能量系统是仿真引擎的关键子系统，负责处理元素微粒/晶球的获取、固定能量恢复、充能效率计算以及角色能量状态管理。系统严格遵循《原神》能量机制，支持站场/后台差异化系数与充能效率加成。

## 1. 核心设计理念

### 1.1 事件驱动架构
能量系统采用事件驱动模式，所有能量变化均通过 `BEFORE_ENERGY_CHANGE` 事件触发，确保与其他系统（武器特效、天赋、共鸣）的松耦合集成。

### 1.2 延迟生效机制
元素微粒/晶球并非即时生效，而是通过 `EnergyDropsObject` 实体在指定帧数后触发能量恢复，模拟真实游戏中的"飞行拾取"延迟。

## 2. 核心组件

### 2.1 EnergySystem 类
**位置**: `core/systems/energy_system.py`

核心系统类，订阅 `BEFORE_ENERGY_CHANGE` 事件并执行能量计算逻辑。

```python
class EnergySystem(GameSystem):
    # 能量获取系数表: {元素匹配类型: {站场状态: 系数}}
    ENERGY_RATES: dict[str, dict[bool, float]] = {
        "同元素": {True: 3.0, False: 1.8},   # 站场: 3.0, 后台: 1.8
        "异元素": {True: 1.0, False: 0.6},   # 站场: 1.0, 后台: 0.6
        "无元素": {True: 2.0, False: 1.2},   # 站场: 2.0, 后台: 1.2 (白色微粒)
    }
```

### 2.2 ElementalEnergy 类
**位置**: `core/mechanics/energy.py`

角色能量状态的数据模型，管理当前能量、能量上限和元素类型。

| 属性 | 类型 | 说明 |
|------|------|------|
| `element` | str | 元素类型（火/水/雷/冰/风/岩/草） |
| `max_energy` | int | 能量上限（角色 Q 技能所需能量） |
| `current_energy` | float | 当前能量（初始 -1 表示未初始化） |

| 方法 | 返回类型 | 说明 |
|------|----------|------|
| `is_full()` | bool | 检查能量是否已满 |
| `is_empty()` | bool | 检查能量是否为空 |
| `clear()` | None | 清空能量（施放元素爆发时调用） |
| `fill()` | None | 填满能量 |
| `gain(amount)` | float | 恢复能量，返回实际恢复值（自动上限截断） |
| `get_progress()` | float | 获取能量进度（0.0~1.0） |
| `get_remaining()` | float | 获取距离满能量还需多少 |

### 2.3 EnergyDropsObject 类
**位置**: `core/entities/energy.py`

能量实体，代表场景中的元素微粒/晶球。存活指定帧数后触发能量恢复事件。

```python
class EnergyDropsObject(BaseEntity):
    def __init__(
        self,
        character,           # 目标角色
        element_energy,      # (元素类型, 基础能量值)
        life_frame=60,       # 存活帧数
        is_fixed=False,      # 是否固定回能
        is_alone=False,      # 是否独立回能
        count=1,             # 微粒数量
    ):
        # 根据 element_energy[1] 值命名:
        # 2 -> "元素微粒"
        # 6 -> "元素晶球"
        # 其他 -> "元素能量"
```

**重要**：`EnergyDropsObject` 必须通过 `ctx.space.register()` 注册到 `CombatSpace`，才能在帧更新时触发 `on_finish()` 生命周期。

### 2.4 EntityFactory.spawn_energy()
**位置**: `core/factory/entity_factory.py`

统一的能量产生接口，由技能系统调用产生微粒/晶球。

```python
@staticmethod
def spawn_energy(
    num: int,              # 微粒数量
    character: Any,        # 目标角色
    element_energy: Any,   # 元素类型与基础值，如 ("水", 2)
    is_fixed: bool = False,# 是否为固定回能
    is_alone: bool = False,# 是否为独立回能
    time: int = 40,        # 延迟帧数（0 表示立即触发）
) -> None:
```

**实现细节**：
- 当 `time > 0`：创建单个 `EnergyDropsObject` 实体并注册到 `CombatSpace`，实体存活结束后触发 `BEFORE_ENERGY_CHANGE` 事件
- 当 `time == 0`：立即发布 `BEFORE_ENERGY_CHANGE` 事件，不走实体生命周期
- 微粒数量 `num` 会合并到 `amount` 中：`(元素, 基础值 × num)`，确保只触发一次事件

## 3. 能量获取机制

### 3.1 能量来源分类

| 类型 | 说明 | 示例 |
|------|------|------|
| **元素微粒** | 技能命中敌人产生，每个 2 点基础能量 | E 技能命中产生 2-3 个微粒 |
| **元素晶球** | 击败敌人掉落，每个 6 点基础能量 | 击杀精英怪掉落 |
| **固定值恢复** | 不受元素/站场/充能效率影响 | 西风武器特效、部分命座效果 |
| **独立恢复** | 队伍共享但按角色独立计算 | 通用机制，预留扩展 |

### 3.2 能量计算公式

```
最终能量 = 微粒基础值 × 元素系数 × 充能效率

元素系数:
┌─────────┬────────┬────────┐
│ 匹配类型 │ 站场   │ 后台   │
├─────────┼────────┼────────┤
│ 同元素   │ 3.0    │ 1.8    │
│ 异元素   │ 1.0    │ 0.6    │
│ 无元素   │ 2.0    │ 1.2    │
└─────────┴────────┴────────┘
```

### 3.3 充能效率计算
**位置**: `core/systems/utils.py` - `AttributeCalculator.get_final_er()`

```python
@staticmethod
def get_final_er(entity) -> float:
    """合算最终元素充能效率 (%)"""
    val = float(entity.attribute_data.get('元素充能效率', 100.0))
    for m in getattr(entity, 'dynamic_modifiers', []):
        if m.stat == '元素充能效率':
            val += m.value
    return val
```

充能效率来源：
- 角色基础属性（默认 100%）
- 武器副属性/特效
- 圣遗物主属性/副词条
- 动态修饰符（技能/武器特效临时加成）

## 4. 事件系统交互

### 4.1 能量相关事件

| 事件类型 | 触发时机 | 数据结构 |
|----------|----------|----------|
| `BEFORE_ENERGY_CHANGE` | 能量变化前 | `{character, amount, is_fixed, is_alone, source_character}` |
| `AFTER_ENERGY_CHANGE` | 能量变化后 | `{character, new_energy, delta, source_type}` |

### 4.2 事件流转

```
技能产球
    ↓
EntityFactory.spawn_energy(num=2, element_energy=("水", 2), time=40)
    ↓
创建 EnergyDropsObject (count=2) 并注册到 CombatSpace
    ↓
CombatSpace 每帧调用 on_frame_update()
    ↓
存活帧数结束 → on_finish() → 发布 BEFORE_ENERGY_CHANGE
    │
    │  事件数据: amount = ("水", 4)  # 2 × 2
    ↓
EnergySystem 处理 → 计算系数，应用充能效率
    ↓
ElementalEnergy.gain() (自动上限检查)
    ↓
发布 AFTER_ENERGY_CHANGE (供其他系统监听)
```

## 5. 代码集成示例

### 5.1 技能产生元素微粒
**示例**：哥伦比娅元素战技

```python
# character/NODKRAI/columbina/skills.py
def spawn_particle(self, source_name: str = "引力涟漪") -> bool:
    """执行产球逻辑。"""
    # 随机决定微粒数量
    rates: tuple[float, float] = MECHANISM_CONFIG["ENERGY_PARTICLE_RATES"]
    num_particles = 1 if random.random() < rates[0] else 2  # 1个:66.67%, 2个:33.33%

    from core.factory.entity_factory import EntityFactory
    EntityFactory.spawn_energy(
        num=num_particles,               # 微粒数量（会合并为单次事件）
        character=self.caster,
        element_energy=("水", 2),        # 水元素微粒，每个 2 点基础能量
        time=40,                         # 约 0.67 秒后生效（60帧=1秒）
    )
    return True
```

### 5.2 武器固定回能
**示例**：西风系列武器

```python
def _on_crit_trigger(self):
    # 通过事件系统恢复固定能量
    self.event_engine.publish(
        GameEvent(
            event_type=EventType.BEFORE_ENERGY_CHANGE,
            frame=current_frame,
            source=self.character,
            data={
                "character": self.character,
                "amount": 6.0,            # 固定 6 点能量
                "is_fixed": True,          # 不受系数影响
            }
        )
    )
```

### 5.3 角色初始化能量
**示例**：角色类 `_setup_character_components()` 方法

```python
# character/NODKRAI/columbina/char.py
def _setup_character_components(self) -> None:
    ...
    # 初始化元素能量（水元素，60点上限）
    self.elemental_energy = ElementalEnergy(element="水", max_energy=60)
```

### 5.4 监听能量变化
**示例**：武器效果监听能量恢复

```python
def _on_energy_change(self, event: GameEvent):
    if event.event_type == EventType.AFTER_ENERGY_CHANGE:
        delta = event.data.get("delta", 0)
        if delta > 0:
            # 触发武器特效
            self._trigger_weapon_effect()
```

## 7. 能量技能基类 (EnergySkill)

**位置**: `core/skills/base.py`

元素爆发技能的基类，提供能量检查和消耗功能：

```python
class EnergySkill(SkillBase):
    def can_cast(self) -> bool:
        """检查能量是否已满"""
        return self.caster.elemental_energy.is_full()

    def consume_energy(self) -> None:
        """消耗并清空施法者的能量"""
        self.caster.elemental_energy.clear()
```

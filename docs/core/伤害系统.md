# Core - 伤害系统 (Damage System) V2.5.5

伤害系统是仿真引擎的核心，负责实现从攻击意图到最终数值结算的全流程逻辑。V2.5.5 架构引入了 **“存储优化审计 (Storage-Optimized Audit)”** 与 **“物理公式对齐 (Physical Formula Alignment)”** 模式，确保了高保真仿真与极低数据库负载。

## 1. 核心设计理念

### 1.1 存储优化策略 (Audit Filtering)
为了支持海量仿真，系统区分了静态检索数据与动态审计数据：
- **[R] (Retrieved)**: 基础属性、角色等级、初始抗性等静态快照。这些数据通过 `AttributeCalculator` 抓取参与计算，但 `audit=False`（不存入 L2 审计表），UI 渲染时通过 `frame_id` 还原。
- **[S] (Stored)**: 技能倍率、随机暴击结果、外部动态 Buff。这些数据 `audit=True`，物理存储在审计链中。

### 1.2 物理公式对齐 (Physical Consistency)
严格遵循《原神》底层防御减免率公式与抗性分段函数，确保仿真精度。

---

## 2. 核心组件

### 2.1 Damage 行为载体
`Damage` 对象封装了攻击的物理契约：
- **倍率向量 (`damage_multiplier`)**: 支持多属性缩放（如艾尔海森的 ATK+EM 混合倍率）。
- **属性缩放 (`scaling_stat`)**: 与倍率向量一一对应的属性名称列表。

### 2.2 DamageContext 审计上下文
作为单次计算的“沙盒”，维护以下核心数据：
- **属性字典 (`stats`)**: 存储计算过程中的实时累积值（用于代数聚合）。
- **审计链 (`audit_trail`)**: 存储 `ModifierRecord` 向量，仅记录标记为 `audit=True` 的变更。

---

## 3. 五阶段伤害流水线 (DamagePipeline)

### 阶段一：准备 (Preparation)
解析攻击标签，锁定目标，触发 `CombatSpace` 空间广播。

### 阶段二：基石 (Foundation) - [R] 静态抓取
1. **面板固化**: 通过 `AttributeCalculator.get_val_by_name` 抓取 `scaling_stat` 指定的所有属性最终值。
2. **契约加载**: 载入技能基础倍率。
3. **初始环境**: 抓取精通与对应元素的伤害加成快照。

### 阶段三：演变 (Evolution) - [S] 动态注入
发布 `BEFORE_CALCULATE` 事件，允许外部系统（武器效果、队友天赋）注入增量 Buff。

### 阶段四：结算 (Resolution) - 系数生成
基于 Stage 3 固化的 stats 执行物理环境结算与随机判定（暴击）。

### 阶段五：聚合 (Synthesis) - 纯函数合算
执行最终的代数合并公式。

## 4. 审计字段详述 (Audit Fields)

根据 V2.5 规范，`DamageContext.stats` 包含以下标准槽位：

| 字段名 | 乘区归属 | 物理含义与计算规则 | 审计策略 |
| :--- | :--- | :--- | :--- |
| **基础属性** | 基础区 | 由 `scaling_stat` 决定的属性快照值（如：2500 攻击力）。 | **[R]** |
| **技能倍率%** | 倍率区 | 技能契约定义的原始百分比（如：80.5%）。 | **[S]** |
| **独立乘区%** | 倍率区 | 对应规范 2.2，执行 `Skill_Mult * (1 + 独立%)`。用于特定机制。 | **[S]** |
| **倍率加值%** | 倍率区 | 申鹤/云堇等提供的直接倍率加算（基础属性 * 加值%）。 | **[S]** |
| **固定伤害值加成**| 核心区 | 基础伤害之外的固定数值加算（如 C6 芙宁娜 HP 转化伤害）。 | **[S]** |
| **伤害加成** | 增伤区 | 对应面板“元素伤害加成”及各类增伤 Buff（如气氛值转化）。 | **[R]+[S]** |
| **暴击乘数** | 暴击区 | 最终合算系数：命中时为 `1 + CD%`，不命中为 `1.0`。 | **[S]** |
| **防御区系数** | 防御区 | 公式：`1 - Def' / (Def' + K)`。其中 `Def'` 受减防/无视影响。 | **[R]+[S]** |
| **抗性区系数** | 抗性区 | 基于目标最终抗性执行分段函数的合算系数。 | **[R]+[S]** |
| **反应加成系数** | 反应区 | 精通转化的反应加成（如：蒸发 +15%）。 | **[S]** |

## 5. 代码集成示例 (Usage Example)

### 示例：芙宁娜召唤物伤害加成注入
芙宁娜召唤物消耗全队血量后，会获得一个属于“增伤区”的动态提升。

```python
def handle_event(self, event: GameEvent):
    if event.event_type == EventType.BEFORE_CALCULATE:
        dmg_ctx = event.data.get("damage_context")
        # 1. 识别目标伤害事件
        if dmg_ctx and dmg_ctx.damage.name == "谢贝蕾妲小姐伤害":
            # 2. 从 Damage.data 获取预计算的加成比例 (如 0.4)
            bonus_ratio = dmg_ctx.damage.data.get("hp_consume_bonus_ratio", 0.0)
            
            if bonus_ratio > 0:
                # 3. 正确使用规范接口注入
                # source: 必填，用于 UI 溯源
                # stat: 对应上述字段详述中的 key
                # value: 传入增量数值 (40.0)
                # op: ADD 代表加法堆叠
                dmg_ctx.add_modifier(
                    source="孤心沙龙-消耗加成", 
                    stat="伤害加成", 
                    value=bonus_ratio * 100.0, 
                    op="ADD"
                )
```

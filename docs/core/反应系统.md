# Core - 反应系统 (Reaction System)

反应系统负责仿真《原神》核心的元素交互逻辑。它基于**附着论 (Elemental Gauge Theory)**，通过事件驱动机制精确模拟元素量消耗、反应优先级、ICD 以及复杂的连锁反应逻辑。

## 1. 核心分层架构

反应逻辑被拆分为三个核心层级，以实现物理判定与逻辑结算的解耦：

### 1.1 状态持有层 (`core.mechanics.aura`)
*   **Aura (元素附着)**: 附着在实体上的元素状态。
*   **Gauge (元素量)**: 严格遵循 1U/2U/4U 标准。
*   **周期性 Tick**: 负责驱动感电（每秒扣减/跳电）与燃烧（每 0.25s 挂火/爆发伤害）的计时逻辑。
*   **来源追踪**: 支持附着来源角色追踪，用于月曜反应的加权伤害计算。

### 1.2 物理交互层 (`core.combat_space`)
*   **物理化副作用**: 某些反应会产生具体的战场实体。
    *   **结晶 (Crystallize)**: 产生 `CrystalShardEntity` (晶片)，角色靠近后自动拾取获得护盾。
    *   **绽放 (Bloom)**: 产生 `DendroCoreEntity` (草原核)，存在 6s 或上限 5 个后自动爆炸。
*   **空间广播**: 扩散 (Swirl) 会在命中点触发 6m 范围的物理广播，将元素传播至周围目标。

### 1.3 逻辑结算层 (`ReactionSystem`)
*   **剧变伤害产生**: 监听 `BEFORE_DAMAGE` 或反应 Tick 事件，生成独立的剧变伤害对象。
*   **受击 ICD 机制**: 严格限制同一目标对同类剧变伤害的接受频率——**0.5s (30帧) 内最多受 2 次伤害**。
*   **月曜反应转换**: 尝试将普通反应转换为月曜反应，触发特殊的月曜伤害机制。

## 2. 反应类别

### 2.1 ReactionCategory 枚举

系统定义了五种反应类别，决定了系统如何应用该反应：

| 类别 | 枚举值 | 说明 | 典型反应 |
|------|--------|------|----------|
| **增幅类** | `AMPLIFYING` | 乘法加成，提升本次伤害倍率 | 蒸发、融化 |
| **加算类** | `ADDITIVE` | 基础伤害加成，固定伤害值加成 | 超激化、蔓激化 |
| **剧变类** | `TRANSFORMATIVE` | 独立伤害，不受攻击力/倍率影响 | 超载、感电、超导、扩散、碎冰、绽放等 |
| **状态类** | `STATUS` | 改变目标状态或生成对象 | 冻结、结晶、燃烧、原激化 |
| **月曜类** | `LUNAR` | 可暴击、无视防御、无增伤加成 | 月绽放、月感电、月结晶 |

## 3. 反应效果详述

| 反应类型 | 类别 | 物理表现 / 副作用 |
| :--- | :--- | :--- |
| **蒸发/融化** | 增幅 | 无实体，直接提升本次伤害倍率（强克制 2.0x，弱克制 1.5x）。 |
| **感电** | 剧变 | 水雷共存，每秒触发一次跳电伤害（倍率 1.2）。可传导至 5m 内水附着目标。 |
| **超载** | 剧变 | 产生火元素范围伤害（倍率 2.75）。 |
| **超导** | 剧变 | 产生冰元素范围伤害（倍率 0.5），降低目标 **40% 物理抗性**，持续 12s。 |
| **扩散** | 剧变 | 造成对应元素伤害（倍率 0.6），并将该元素传播至 6m 内的其他实体。 |
| **结晶** | 状态 | 掉落对应元素晶片，拾取后获得 15s 属性匹配护盾。**火/水/雷/冰结晶共享 1 秒冷却**。 |
| **绽放** | 剧变 | 生成草原核（倍率 2.0）。受火/雷攻击触发烈绽放（3.0）/超绽放（3.0）。 |
| **激化** | 状态 | 产生激元素。后续雷/草攻击享受固定伤害值加成（超激化/蔓激化）。 |
| **月绽放** | 月曜 | 保持原有绽放机制，额外触发草露资源恢复。详见 [月曜反应文档](月曜反应.md)。 |
| **月感电** | 月曜 | 取消普通感电状态，生成雷暴云实体进行周期性攻击。详见 [月曜反应文档](月曜反应.md)。 |
| **月结晶** | 月曜 | 不生成晶片，生成月笼攻击实体，每 3 次触发谐奏攻击。**无冷却限制**。详见 [月曜反应文档](月曜反应.md)。 |

## 4. 元素附着规则 (Gauge Theory)

### 4.1 Gauge 数据结构

```python
@dataclass
class Gauge:
    element: Element           # 元素类型
    u_value: float            # 初始元素量 (1U, 2U, 4U)
    max_gauge: float          # 最大附着量 (0.8 * u_value)
    current_gauge: float      # 当前剩余附着量
    decay_rate: float         # 衰减速率 (每秒扣除量)
    source_character: Any     # 附着来源角色 (月曜反应追踪)
    source_time: float        # 附着时间戳
```

### 4.2 刷新规则 (Refresh)

当实体已带有同元素附着时：
- **元素量**: 取当前剩余量与新附着量中的**较大值**。
- **衰减速率**: 强制覆盖为**新附着量的衰减速率**（关键：1U 无法覆盖 2U 的量，但会加快其消失）。
- **来源追踪**: 如果新附着有来源角色，更新来源信息。

### 4.3 反应消耗 (Tax)

- **增幅反应**: 存在 2.0x (强克制) 和 0.5x (弱克制) 的消耗比。
    - 水触发火（蒸发）：消耗 2.0x 火元素
    - 火触发水（蒸发）：消耗 0.5x 水元素
    - 火触发冰（融化）：消耗 2.0x 冰元素
    - 冰触发火（融化）：消耗 0.5x 火元素
- **风/岩反应**: 固定的 0.5x 消耗比，且作为后置元素时永不残留。
- **草系反应**: 绽放消耗比 1.0x。

### 4.4 附着来源追踪

`Gauge.source_character` 字段用于月曜反应的加权伤害计算：

- 记录造成附着的角色来源
- 月感电/月结晶时，所有参与附着的角色都参与伤害组分计算
- 最终伤害采用加权求和：`最高 + 次高÷2 + 其余之和÷12`

## 5. 结晶反应冷却机制

### 5.1 全局冷却规则

**火/水/雷/冰结晶共享 1 秒冷却**，按被消耗的元素类型分别计时：

```python
# 冷却追踪
CRYSTALLIZE_COOLDOWN: float = 1.0  # 秒

# key: 被结晶消耗的元素类型 (Element.HYDRO/PYRO/ELECTRO/CRYO)
# value: 上次触发结晶的时间戳（帧数）
_crystallize_cooldowns: dict[Element, int] = {}
```

### 5.2 冷却检查流程

```
岩攻击命中 → 检测附着元素 → 判定结晶类型
    │
    ▼
检查该元素类型的结晶冷却
    │
    ├─ 在冷却中 → 标记 is_cooldown_skipped = True
    │              └─ 尝试转换为月结晶（月结晶无冷却）
    │
    └─ 不在冷却中 → 正常触发结晶
                    └─ 更新冷却时间戳
```

### 5.3 月结晶特殊处理

**月结晶无冷却限制**：

- 普通结晶冷却中的水结晶可转换为月结晶
- 转换后 `is_cooldown_skipped` 被设为 `False`，绕过冷却检查
- 月结晶不消耗普通结晶的冷却次数

## 6. 剧变反应受击 ICD

### 6.1 机制说明

剧变反应伤害有独立的受击限制，防止同一目标承受过高频率的反应伤害：

**规则**：同一目标对同类剧变伤害，**0.5s (30帧) 内最多受 2 次**。

```python
# 受击 ICD 配置
_REACTION_ICD_WINDOW = 30    # 30帧 = 0.5秒
_REACTION_MAX_HITS = 2       # 窗口内最大受击次数

# 追踪记录
# Key: (目标实体ID, 反应类型) -> [最后重置帧, 当前窗口内受击次数]
_target_reaction_records: dict[tuple[int, ElementalReactionType], list[int]] = {}
```

### 6.2 与附着 ICD 的区别

| 特性 | 附着 ICD | 受击 ICD |
|------|----------|----------|
| 作用对象 | 攻击者 | 受击者 |
| 控制内容 | 是否产生附着 | 是否承受伤害 |
| 时间窗口 | 通常 2.5s | 固定 0.5s |
| 触发限制 | 命中次数限制 | 受击次数限制 |
| 共享范围 | 同 icd_group 共享 | 同目标同反应类型共享 |

### 6.3 适用范围

受击 ICD 适用于所有剧变反应伤害：
- 超载、感电、超导、扩散、碎冰
- 绽放、超绽放、烈绽放

## 7. 月曜反应系统

月曜反应是元素反应的**第三种类别**，与增幅反应、剧变反应并列。详细机制参见 [月曜反应文档](月曜反应.md)。

### 7.1 触发条件

| 条件 | 描述 |
|------|------|
| 队伍配置 | 队伍中有特定角色（月兆天赋持有者） |
| 触发来源 | 仅角色触发的反应可转化为月曜反应 |
| 敌人/环境 | 触发的反应仍为原反应 |

### 7.2 月曜伤害特性

| 特性 | 月曜伤害 | 直接伤害 | 剧变伤害 |
|------|:--------:|:--------:|:--------:|
| 可暴击 | ✅ | ✅ | ❌ |
| 享受增伤加成 | ❌ | ✅ | ❌ |
| 无视防御 | ✅ | ❌ | ❌ |
| 产生附着 | ❌ | ✅ | ❌ |

### 7.3 三种月曜反应

| 反应 | 原反应 | 特殊效果 |
|------|--------|----------|
| **月绽放** | 绽放 | 保持原有机制，额外生成草露资源 |
| **月感电** | 感电 | 取消感电状态，生成雷暴云实体 |
| **月结晶** | 结晶(水) | 不生成晶片，生成月笼攻击实体 |

### 7.4 反应转换流程

```
原反应触发
    │
    ▼
_try_convert_to_lunar()
    │
    ├─ 检查触发源是否为角色
    ├─ 获取队伍成员
    └─ 检查是否有对应月曜角色
        │
        ▼
    满足条件 → 转换为月曜反应
    │
    ▼
_apply_lunar_reaction_effect()
    │
    ├─ 月绽放 → 生成草原核 + 触发草露恢复
    ├─ 月感电 → 生成/刷新雷暴云
    └─ 月结晶 → 生成月笼 + 累积谐奏计数
```

## 8. 周期性反应机制

### 8.1 感电 (Electro-Charged)

| 属性 | 描述 |
|------|------|
| 攻击间隔 | 每 1 秒结算 1 次 |
| 元素消耗 | 每次消耗 0.4U 水元素 + 0.4U 雷元素 |
| 传导范围 | 5 米 |
| 传导条件 | 同阵营 + 有水附着 |
| Tick 倍率 | 1.2 |

### 8.2 燃烧 (Burning)

| 属性 | 描述 |
|------|------|
| 攻击间隔 | 每 0.25 秒结算 1 次 |
| 元素消耗 | 每秒 0.4U 草/激元素 |
| 自挂火 | 保持火附着 |
| 范围伤害 | 半径 1 米 |
| Tick 倍率 | 0.25 |

## 9. 开发指南

### 9.1 受击限制

所有剧变伤害均通过 `_generate_transformative_damage` 产生，自动享受受击 ICD 保护。

### 9.2 自定义反应

若需增加特殊反应：
1. 在 `core/systems/contract/reaction.py` 定义 `ElementalReactionType` 枚举
2. 在 `REACTION_CLASSIFICATION` 中注册反应类别
3. 在 `ReactionSystem` 中添加对应处理分支

### 9.3 月曜反应开发

开发月曜相关角色时：
- 使用 `Gauge.create()` 时传入 `source_character` 参数
- 月曜转换由 `ReactionSystem._try_convert_to_lunar()` 自动处理
- 月曜伤害通过 `_apply_lunar_damage()` 应用

### 9.4 结晶冷却注意事项

- 冷却按**被消耗的元素类型**分别计时
- 月结晶转换后不消耗普通结晶的冷却次数
- 月结晶生成后会重置该元素类型的结晶冷却

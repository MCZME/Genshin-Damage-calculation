# Core - 模拟上下文 (SimulationContext)

`SimulationContext` 是整个仿真引擎的“核心容器”与“真理来源”。它持有当前模拟的所有状态，并负责协调各个子系统的运行。

## 1. 核心设计
为了支持高性能的并行模拟并彻底隔离状态，本项目摒弃了全局变量。

### 1.1 ContextVar 隔离
所有的状态都封装在 `SimulationContext` 实例中，并通过 Python 的 `ContextVar` 实现线程/协程安全的访问。通过 `get_context()` 获取的始终是当前调用链所属的模拟环境。

## 2. 关键组件
- **`current_frame`**: 全局帧计数器。
- **`space`**: [NEW] `CombatSpace` 实例，负责全量实体的空间坐标与碰撞检索。
- **`event_engine`**: 层级事件引擎实例。
- **`system_manager`**: 系统管理器，负责生命周期内各 `GameSystem`（如 `DamageSystem`）的装配。
- **`team`**: 当前参与模拟的队伍实体容器。
- **`target`**: 模拟的目标（敌人）引用（保持兼容）。

## 3. 使用规范

### 3.1 创建与初始化
通常使用工厂函数创建，它会自动装配所有核心业务系统：
```python
from core.context import create_context

ctx = create_context() 
```

### 3.2 作用域管理
推荐使用 `with` 语句确保上下文在模拟结束后正确重置：
```python
with ctx:
    # 在此作用域内，get_context() 将返回 ctx
    simulator.run()
```

### 3.3 获取上下文
在任何业务逻辑（如技能、效果）中获取当前环境：
```python
from core.context import get_context

ctx = get_context()
print(f"当前帧: {ctx.current_frame}")
```

## 4. 生命周期
1. **装配 (`create_context`)**: 扫描注册表，初始化所有 System。
2. **驱动 (`advance_frame`)**: 
    - 帧数自增。
    - 驱动 `space.update()` 完成全场景实体的物理更新。
3. **清理 (`reset`)**: 移除所有实体、清空事件队列，释放资源。

### 4.1 全场景快照 (`take_snapshot`)
用于 UI 展示或持久化。
- **功能**: 遍历 `CombatSpace` 中的所有实体，调用其 `export_state()` 方法。
- **返回**: 包含当前帧数、全局状态及所有实体数据的纯字典。

# 实体系统架构 (Entity System Architecture)

## 1. 概述

实体系统是原神战斗仿真引擎的核心架构之一，负责管理仿真世界中所有具有物理存在和生命周期的对象。V2.5 版本重构后，实体系统采用分层继承设计，为不同类型的实体提供最小化的属性集和扩展点。

## 2. 实体层级架构

```
BaseEntity (基础实体)
│   ├── 生命周期管理
│   ├── 空间属性：pos, facing, hitbox, faction
│   ├── 帧驱动接口
│   └── 状态导出
│
├── CombatEntity (战斗实体)
│   │   ├── 元素附着：AuraManager
│   │   ├── ICD 管理：ICDManager
│   │   ├── 效果管理：active_effects, shield_effects
│   │   ├── 属性修饰符：dynamic_modifiers
│   │   └── 子类：Character, Target, 召唤物, 元素产物
│   │
│   └── 特有能力
│       ├── 接收伤害：handle_damage()
│       ├── 接收治疗：heal()
│       ├── 元素附着：apply_elemental_aura()
│       └── 属性修饰：add_modifier(), remove_modifier()
│
├── AttackEntity (攻击实体)
│   │   ├── 承载一次攻击
│   │   ├── 多种触发模式
│   │   └── 自动索敌
│   │
│   └── 典型应用
│       ├── 投射物（追踪箭矢）
│       ├── 延迟爆炸（炸弹）
│       └── 命中帧攻击（多段攻击）
│
└── SceneEntity (场景实体)
    │   ├── 长期存在场景中
    │   ├── 区域检测
    │   └── 周期触发
    │
    └── 典型应用
        ├── 场地效果（伤害区域）
        ├── Buff 区域（增益光环）
        └── 触发区域（机关）
```

## 3. BaseEntity - 基础实体

### 3.1 设计理念

`BaseEntity` 是所有实体的基类，提供最核心的功能：
- **生命周期管理**：统一的创建、活跃、销毁状态流转
- **空间属性**：所有实体都具备位置、朝向、碰撞盒和阵营
- **帧驱动**：统一的 `on_frame_update()` 入口，子类重写 `_perform_tick()`

### 3.2 生命周期状态

```python
class EntityState(Enum):
    INIT = auto()        # 初始化（创建后立即进入 ACTIVE）
    ACTIVE = auto()      # 活跃中（参与帧更新）
    FINISHING = auto()   # 正在结束（执行清理逻辑）
    DESTROYED = auto()   # 已彻底销毁
```

状态流转：
```
创建 → INIT → ACTIVE → (life_frame 到期或主动 finish) → FINISHING → on_finish() → DESTROYED
```

### 3.3 核心属性

| 属性 | 类型 | 说明 |
|------|------|------|
| `entity_id` | int | 全局唯一实体 ID（自动分配） |
| `name` | str | 实体名称 |
| `life_frame` | float | 生存帧数（默认 inf 永久） |
| `current_frame` | int | 当前已存活帧数 |
| `state` | EntityState | 生命周期状态 |
| `pos` | list[float] | 位置坐标 [x, z, y] |
| `facing` | float | 朝向角度（度） |
| `hitbox` | tuple[float, float] | 碰撞盒 (radius, height) |
| `faction` | Faction | 阵营标识 |

### 3.4 帧驱动流程

```python
def on_frame_update(self) -> None:
    """帧驱动入口（不要重写此方法）"""
    if self.state == EntityState.FINISHING:
        self.finish()
        return

    if self.state != EntityState.ACTIVE:
        return

    self.current_frame += 1
    if self.current_frame >= self.life_frame:
        self.finish()
        return

    self._perform_tick()  # 子类重写此方法

def _perform_tick(self) -> None:
    """业务逻辑钩子（子类重写）"""
    pass
```

### 3.5 阵营系统

```python
class Faction(Enum):
    PLAYER = auto()    # 玩家/友方（角色、召唤物）
    ENEMY = auto()     # 敌人/敌对
    NEUTRAL = auto()   # 中立/环境物（草原核、结晶晶片）
```

阵营用于：
- 空间查询时的目标过滤
- 伤害广播时的阵营判定
- 元素广播的目标选择

## 4. CombatEntity - 战斗实体

### 4.1 设计理念

`CombatEntity` 是需要参与战斗逻辑的实体基类。相比 `BaseEntity`，它增加了：
- **元素系统**：元素附着管理（Aura）和附着冷却（ICD）
- **效果系统**：可挂载/移除的效果（Buff/Debuff/护盾）
- **属性系统**：动态属性修饰符（用于伤害计算）

### 4.2 核心属性

| 属性 | 类型 | 说明 |
|------|------|------|
| `aura` | AuraManager | 元素附着管理器 |
| `icd_manager` | ICDManager | 附着冷却管理器 |
| `active_effects` | list[Any] | 当前激活的效果列表 |
| `shield_effects` | list[ShieldEffect] | 当前激活的护盾列表 |
| `dynamic_modifiers` | list[ModifierRecord] | 动态属性修饰符 |
| `custom_metrics` | dict[str, float] | 通用机制指标 |

## 5. AttackEntity - 攻击实体

### 5.1 设计理念

`AttackEntity` 是轻量级的攻击载体，用于承载一次攻击并在满足条件时释放。它**不继承 `CombatEntity`**，因为攻击实体：
- 不需要接收伤害
- 不需要元素附着
- 不需要效果管理
- 只需要基本的空间属性和阵营

### 5.2 触发类型

```python
class AttackTriggerType(Enum):
    ON_FINISH = auto()      # 生命周期结束时触发
    ON_HIT_FRAME = auto()   # 到达指定命中帧时触发
    ON_PROXIMITY = auto()   # 靠近目标时触发（碰撞检测）
    TRACKING = auto()       # 追踪模式（移动追踪目标，碰撞时触发）
```

### 5.3 索敌模式

```python
class TargetingMode(Enum):
    CLOSEST = auto()    # 最近目标
    RANDOM = auto()     # 随机目标
    FIXED = auto()      # 固定目标（需要预设 tracking_target）
```

### 5.4 核心属性

| 属性 | 类型 | 说明 |
|------|------|------|
| `damage` | Damage | 承载的伤害对象 |
| `source` | Any | 伤害来源角色 |
| `trigger_type` | AttackTriggerType | 触发类型 |
| `targeting_mode` | TargetingMode | 索敌模式 |
| `tracking_speed` | float | 追踪速度（TRACKING 模式） |
| `tracking_target` | Any | 追踪目标 |
| `has_attacked` | bool | 是否已释放攻击（确保只触发一次） |

### 5.5 使用示例

```python
# 延迟爆炸
explosion = AttackEntity(
    name="延迟爆炸",
    damage=Damage(element=(Element.PYRO, 1.0), damage_multiplier=(3.0,)),
    source=character,
    pos=(5.0, 5.0, 0.0),
    hitbox=(3.0, 3.0),
    life_frame=60,  # 1 秒后爆炸
    trigger_type=AttackTriggerType.ON_FINISH,
)
ctx.space.register(explosion)

# 追踪投射物
projectile = AttackEntity(
    name="追踪箭矢",
    damage=Damage(element=(Element.HYDRO, 1.0), damage_multiplier=(1.5,)),
    source=character,
    pos=character.pos.copy(),
    hitbox=(0.3, 0.3),
    life_frame=120,
    trigger_type=AttackTriggerType.TRACKING,
    tracking_speed=0.5,
    tracking_target=enemy,
)
ctx.space.register(projectile)

# 多段攻击（命中帧模式）
multi_attack = AttackEntity(
    name="三连击",
    damage=Damage(...),
    source=character,
    pos=character.pos.copy(),
    trigger_type=AttackTriggerType.ON_HIT_FRAME,
)
multi_attack.set_hit_frames([15, 30, 45])  # 第 15, 30, 45 帧触发
ctx.space.register(multi_attack)
```

## 6. SceneEntity - 场景实体

### 6.1 设计理念

`SceneEntity` 用于需要在场景中长期存在并承载特殊逻辑的实体。它**不继承 `CombatEntity`**，因为场景实体：
- 通常不需要接收伤害（如 Buff 区域）
- 不需要元素附着
- 需要区域检测和周期性触发能力

### 6.2 核心属性

| 属性 | 类型 | 说明 |
|------|------|------|
| `owner` | Any | 所属实体（如创建者角色） |
| `tick_interval` | int | 周期触发间隔（帧） |
| `detection_radius` | float | 区域检测半径 |
| `affect_factions` | list[Faction] | 影响的阵营列表 |
| `tick_timer` | int | 当前计时器 |
| `_entities_in_range` | set[int] | 区域内实体 ID 集合 |

### 6.3 区域检测

场景实体每帧检测区域内的实体变化，并触发相应事件：

```python
def _check_entities_in_range(self) -> None:
    # 1. 获取当前区域内的所有实体
    current_entities = set()
    for faction in self.affect_factions:
        entities = self.ctx.space.get_entities_in_range(...)
        current_entities.update(e.entity_id for e in entities)

    # 2. 检测进入事件
    for eid in current_entities - self._entities_in_range:
        entity = self._find_entity_by_id(eid)
        self._on_entity_enter(entity)

    # 3. 检测离开事件
    for eid in self._entities_in_range - current_entities:
        entity = self._find_entity_by_id(eid)
        self._on_entity_exit(entity)

    # 4. 更新状态
    self._entities_in_range = current_entities
```

### 6.4 可重写钩子

```python
class SceneEntity(BaseEntity):
    def _on_entity_enter(self, entity: Any) -> None:
        """实体进入区域（子类重写）"""
        pass

    def _on_entity_exit(self, entity: Any) -> None:
        """实体离开区域（子类重写）"""
        pass

    def _on_entity_stay(self, entity: Any) -> None:
        """实体持续在区域内（子类重写，每帧调用）"""
        pass

    def _on_tick(self) -> None:
        """周期触发（子类重写）"""
        pass

    def refresh(self) -> None:
        """刷新存在时间（延长生命周期）"""
        self.current_frame = 0
```

### 6.5 使用示例

```python

# Buff 区域
class BuffZone(SceneEntity):
    def __init__(self, owner, buff_factory, pos, radius, duration):
        super().__init__(
            name="增益区域",
            pos=pos,
            hitbox=(radius, radius),
            detection_radius=radius,
            affect_factions=[Faction.PLAYER],
            owner=owner,
            life_frame=duration,
        )
        self.buff_factory = buff_factory
        self._buffed_entities = {}

    def _on_entity_enter(self, entity):
        buff = self.buff_factory(entity)
        buff.apply()
        self._buffed_entities[entity.entity_id] = buff

    def _on_entity_exit(self, entity):
        if entity.entity_id in self._buffed_entities:
            self._buffed_entities[entity.entity_id].remove()
            del self._buffed_entities[entity.entity_id]

    def on_finish(self):
        # 销毁时清除所有 Buff
        for buff in self._buffed_entities.values():
            buff.remove()
        super().on_finish()
```

### 7.3 伤害广播

`broadcast_damage` 只对 `CombatEntity` 类型分发伤害：

```python
def broadcast_damage(self, attacker: CombatEntity, damage: Damage) -> None:
    targets = self.get_entities_in_range(...)
    for t in targets:
        if isinstance(t, CombatEntity):
            t.handle_damage(damage)
```

## 9. 最佳实践

### 9.1 选择正确的基类

| 需求 | 推荐基类 |
|------|----------|
| 需要接收伤害 | `CombatEntity` |
| 需要元素附着 | `CombatEntity` |
| 需要挂载效果/修饰符 | `CombatEntity` |
| 只需要承载一次攻击 | `AttackEntity` |
| 需要区域检测 | `SceneEntity` |
| 需要周期性触发 | `SceneEntity` |
| 只需要基本位置属性 | `BaseEntity` |

### 9.2 重写指南

```python
class MyEntity(BaseEntity):
    def _perform_tick(self) -> None:
        """每帧业务逻辑"""
        super()._perform_tick()  # 建议调用父类
        # 自定义逻辑...

    def on_finish(self) -> None:
        """销毁前清理"""
        # 清理资源...
        super().on_finish()  # 必须调用父类

    def export_state(self) -> dict[str, Any]:
        """导出状态"""
        base = super().export_state()
        base.update({"my_field": self.my_field})
        return base

    def export_static_data(self) -> dict[str, Any]:
        """导出静态配置"""
        data = super().export_static_data()
        data["entity_type"] = "MY_ENTITY"
        # 添加特有属性...
        return data
```

# Core - 效果系统 (Effect System)

效果系统处理仿真中的增益、减益、护盾、持续伤害及特殊状态逻辑，采用事件驱动架构与模板方法模式。

## 1. 模块结构

```
core/effect/
├── base.py      # 效果基类、堆叠规则枚举
└── common.py    # 通用效果类（护盾、属性修改、天赋、命座、月兆）

core/entities/base_entity.py   # 效果容器与查询接口
core/persistence/projector.py  # 效果生命周期持久化
```

## 2. 继承层次

## 3. BaseEffect 核心属性

```python
class BaseEffect(ABC):
    def __init__(
        self,
        owner: Any,                         # 效果持有者
        name: str,                          # 效果名称
        duration: float = 0,                # 剩余帧数 (float('inf') 为永久)
        stacking_rule: StackingRule = StackingRule.REFRESH,
    ):
        self.is_active = False              # 激活状态
        self.instance_id: int = 0           # 全局唯一实例 ID
```

### 堆叠规则

| 规则 | 行为 | 典型场景 |
|------|------|----------|
| `REFRESH` | 刷新持续时间 | 大多数增益效果 |
| `ADD` | 增加层数 | 暴击率叠层、气氛值累计 |
| `INDEPENDENT` | 独立共存 | 可叠加的 Debuff |

### 生命周期

```
apply() → is_active → remove()
    │          │          │
    ▼          ▼          ▼
on_apply() on_tick()  on_remove()
```

| 钩子 | 触发时机 | 用途 |
|------|----------|------|
| `on_apply()` | 应用时 | 订阅事件、添加修饰符 |
| `on_remove()` | 移除时 | 取消订阅、清理修饰符 |
| `on_tick(target)` | 每帧 | 持续伤害、计时器 |
| `on_stack_added(other)` | 叠加时 | 增加层数 |

## 6. 属性修改路径

| 类型 | 方式 | 场景 |
|------|------|------|
| 持久修改 | `owner.add_modifier()` | 持续时间内的属性加成 |

## 7. 事件驱动

```python
class MyEffect(BaseEffect):
    def on_apply(self):
        self.owner.event_engine.subscribe(EventType.BEFORE_CALCULATE, self)

    def on_remove(self):
        self.owner.event_engine.unsubscribe(EventType.BEFORE_CALCULATE, self)

    def handle_event(self, event):
        if event.event_type == EventType.BEFORE_CALCULATE:
            dmg_ctx = event.data.get("damage_context")
            dmg_ctx.add_modifier(source="效果", stat="伤害加成", value=50.0)
```

### 关键事件

| 事件类型 | 用途 |
|----------|------|
| `BEFORE_CALCULATE` | 伤害计算前注入增伤 |
| `AFTER_DAMAGE` | 伤害后触发特效 |
| `BEFORE_HEAL` / `AFTER_HEAL` | 治疗相关 |
| `BEFORE_HURT` / `AFTER_HURT` | 受伤相关 |
| `ON_EFFECT_ADDED` / `ON_EFFECT_REMOVED` | 效果生命周期 |

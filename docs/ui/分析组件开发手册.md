# 分析组件开发手册 (V6.8 Pro)

> 本文档定义了分析磁贴（Tile）在 V6.8 模块化网格系统下的开发规范与技术契约。

---

## 磁贴通用规范 (General Contract)

### 1. 物理规格 (Grid Specs)
磁贴尺寸必须在 `tile_factory` 中声明 **`grid_size` (W, H)**。
*   **W (Width Units)**: 占据的水平单元格数量（每个单元格 **160px**）。
*   **H (Height Units)**: 占据的垂直单元格数量。

### 2. 声明式开发契约 (Declarative V3)
磁贴内部实现必须完全符合 Flet V3 声明式标准：
*   **装饰器**: `render()` 方法必须使用 `@ft.component` 装饰。
*   **状态管理**: 使用 Hooks (`ft.use_state`, `ft.use_effect`) 管理磁贴内部状态（如加载态、局部悬浮态）。
*   **严禁操作**: 严禁直接调用 `self.update()`。UI 的变更应完全由状态驱动。

### 3. 视觉基因契约 (Color Echoing)
每个磁贴类必须在 `__init__` 中显式定义以下属性，以便 `TileContainer` 进行自动视觉装配：
*   **`self.theme_color`**: 磁贴的主题色（如 `#D3BC8E` 琥珀金）。
*   **`self.gradient_top`**: 背景渐变的起始色（通常为主题色的极深 Tint）。

## 容器组件：TileContainer (V4.5 Pro)

所有磁贴内容均由 `TileContainer` 统一包装，具备以下工业级特性：
1.  **1.5px 抗锯齿边框**: 配合微投影，确保边缘锐利度。
2.  **三层空间模型**:
    *   **Identity Bar**: 包含呼吸指示灯和标题。
    *   **Action Cluster**: 鼠标悬浮时显现的微型按钮组（设置、展开、关闭）。
3.  **元数据页脚**: 自动展示磁贴的唯一实例 ID。

## 核心磁贴实现参考

### 全局战报看板 (SummaryTile)
*   **规格**: **2x1 单元格**。
*   **布局**: 2x2 极简指标阵列，数值字体大小 `22` (W900)。

### 伤害分布脉冲图 (DamageDistTile)
*   **规格**: **4x2 单元格**。
*   **特性**: 沉浸式布局，剥离所有浮动面板，仅保留 Canvas 本身。
*   **渲染**: 使用 688px (4单元格宽) 物理对齐的坐标系。

## 开发流程
1.  在 `ui/components/analysis/` 下创建新磁贴类，继承 `AnalysisTile`。
2.  定义 `theme_color` 与 `gradient_top`。
3.  实现 `@ft.component` 装饰的 `render()` 方法。
4.  在 `ui/views/analysis_view.py` 的 `tile_factory` 中注册该磁贴及其默认 `grid_size`。

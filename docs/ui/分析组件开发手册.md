# 分析组件开发手册 (V9.1 MVVM)

> 本文档定义了分析磁贴（Tile）在 V9.1 MVVM 架构下的开发规范、数据订阅契约与技术实现路径。

---

## 磁贴三层解耦架构 (3-Layer Tile Architecture)

自 V9.1 起，每个磁贴组件内部遵循微型 MVVM 模式，确保渲染、逻辑与数据高度解耦：

1.  **磁贴视图 (Tile View)**: 位于 `ui/components/analysis/`。
    - 纯声明式渲染组件（`@ft.component`）。
    - 仅持有对 Tile VM 的引用，通过绑定 VM 属性进行展示。
2.  **磁贴视图模型 (Tile ViewModel)**: 位于 `ui/view_models/analysis/tile_vms/`。
    - 核心职责：**数据适配 (Data Mapping)**。
    - 将 `DataSlot` 中的原始业务数据转换为 UI 可直接消费的格式（如 `fch.LineChartData`）。
    - 处理磁贴内部的局部交互（如点击坐标、局部缩放）。
3.  **数据服务 (Data Service)**: 位于 `ui/services/analysis_data_service.py`。
    - 提供响应式 `DataSlot` 订阅机制。

## 磁贴通用规范 (General Contract)

### 1. 物理规格 (Grid Specs)
磁贴尺寸必须在 `analysis_view.py` 的 `tile_factory` 中显式声明 **`grid_size` (W, H)**。
*   **W (Width Units)**: 占据的水平单元格数量（单元格宽 **160px**）。
*   **H (Height Units)**: 占据的垂直单元格数量（单元格高 **160px**）。
*   **标准规格**:
    - **SummaryTile**: 2x1
    - **DPSChartTile**: 2x2
    - **DamageDistributionTile**: 4x2
    - **CharacterStatsTile**: 2x2

### 2. 声明式开发契约 (Declarative Hooks)
磁贴内部必须完全符合 Flet Hooks 声明式标准：
*   **组件定义**: 使用 `@ft.component` 装饰 `build()` 方法（或作为函数组件）。
*   **状态订阅**: 磁贴初始化时应调用 `data_service.subscribe(key, instance_id)`。
*   **视图模型绑定**: 磁贴组件通过构造函数接收其对应的 `TileViewModel` 实例。

### 3. 视觉基因契约 (Visual DNA)
磁贴类需定义以下属性，供 `TileContainer` 实现自动化装配：
*   **`self.theme_color`**: 磁贴的主题色（如：`#D3BC8E` 琥珀金）。
*   **`self.gradient_top`**: 背景渐变起始色。
*   **`self.instance_id`**: 由 ViewModel 分配的唯一标识符。

## 数据驱动模型 (Data-Driven Model)

### 1. DataSlot 机制
磁贴 VM 负责监听 `DataSlot`：
```python
@property
def loading(self) -> bool:
    slot = self.data_service.get_cached("dps")
    return slot.loading if slot else True

@property
def chart_data(self) -> list[fch.LineChartData]:
    # 将 DataSlot.data 转换为 LineChartData 列表
    ...
```

### 2. 下钻交互 (Drill-Down)
支持下钻的图表需接受 `on_drill_down` 回调：
- **触发场景**: 点击坐标点或柱状图柱体。
- **传递参数**: `point: dict` (包含 `frame`, `event_id` 等)。

## 开发流程 (Standard Workflow)

1.  **创建 ViewModel**: 在 `ui/view_models/analysis/tile_vms/` 创建 `my_tile_vm.py`。
    - 继承 `@ft.observable`。
    - 实现从 `DataSlot` 到 UI 数据结构的转换 Getter。
2.  **创建 View 组件**: 在 `ui/components/analysis/` 下创建磁贴渲染类。
    - 构造函数接收 `vm: MyTileViewModel`。
    - 实现 `@ft.component` 装饰的 `render()` 方法，并绑定 `vm` 属性。
3.  **注册磁贴**:
    - 在 `ui/views/analysis_view.py` 的 `_create_tile_factory` 中添加分支。
    - **实例化 VM**: 首先创建 `TileViewModel` 实例。
    - **实例化 View**: 将 VM 注入磁贴组件并返回。
    - 定义其默认 `grid_size`。

## 技术注意事项
- **引用计数**: `DataService` 会自动管理 `DataSlot` 的引用计数。
- **抗锯齿**: 内部绘图（Canvas）应考虑 `1.5px` 边框偏移，避免视觉溢出。
- **渲染性能 (Memoization)**: 
    - 对于复杂的 Canvas 绘制，推荐将绘图逻辑封装在独立的 `IsolatedCanvas` 类或函数中。
    - 使用 `ft.use_memo` 锁定 Canvas 实例。
    - 仅当关键布局参数（如 `global_peak`, `total_frames`）或底层数据对象引用发生变更时，才允许重新实例化 Canvas。

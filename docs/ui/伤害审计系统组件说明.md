# UI - 伤害审计系统组件说明 (DamageAuditInspector)

伤害审计系统在 UI 层基于 Material Design 3 (MD3) 规范实现，旨在通过直观的视觉反馈和流畅的交互，让用户能够精准回溯每一笔伤害的构成。

## 1. 组件概述 (`DamageAuditInspector`)

`DamageAuditInspector` 是一个高保真的详情观察器，通常嵌套在战术分析视图的 `FloatingDrawer` 容器中。

### 1.1 视觉规范 (Theme Specification)
组件严格遵循 MD3 色彩角色分配，确保在深色模式下的专业感：

| 元素 | 颜色值 | MD3 角色 | 用途 |
| :--- | :--- | :--- | :--- |
| **背景 (Surface)** | `#2B2738` | Surface | 容器主体背景 |
| **卡片 (Container)** | `#322D41` | Surface Variant | 审计明细条目的背景 |
| **高亮 (Primary)** | `#D1A2FF` | Primary | 关键数值、激活态 Chip |
| **文本 (Main)** | `#E5E1E6` | On Surface | 正文与主要数值 |
| **标签 (Label)** | `#ABA6B5` | On Surface Variant | 辅助说明文本 |

## 2. 交互流程：异步下钻加载 (V9.1 MVVM)

组件不再直接操作适配器，而是通过 `AnalysisViewModel` 与 `AnalysisDataService` 协同完成数据流转：

1.  **事件触发**: 当用户在分布图或列表中点击具体伤害事件。
2.  **VM 调度**: `AnalysisViewModel.handle_drill_down` 被调用，触发 `load_audit_detail(event_id)`。
3.  **数据订阅**: `AnalysisDataService` 将加载状态写入 `DataSlot("audit")` 并触发异步 Fetch。
4.  **响应式重绘**: `FloatingDrawer` 容器检测到 `audit_slot.data` 变更，利用 `AnimatedSwitcher` 平滑展示审计明细。

## 3. 核心视图模块

### 3.1 公式拆解区 (Multiplier Chain)
采用双行布局模拟真实的伤害计算公式：
- **布局结构**: 每个乘区（如基础、倍率、增伤）封装为 MD3 `FilterChip` 风格的组件。
- **状态反馈**: 每一个 Chip 均可点击，点击后下方明细列表将自动定位到对应的乘区来源记录。

### 3.2 审计明细列表 (Audit Trail List)
- **MD3 ListTile 风格**: 每一条增益记录（如“班尼特-美妙旅程”）使用圆角卡片展示。
- **视觉引导**: 卡片左侧设有 `Leading Color Strip`，通过颜色区分属性类别（如红色代表攻击相关，紫色代表增伤相关）。
- **数据胶囊**: 右侧使用 Tonal 容器包裹数值（如 `+15307`），突出展示该步骤对最终结果的影响。

## 4. 响应式与性能优化
- **虚拟列表**: 当审计明细超过 20 条时，自动启用虚拟滚动以保证滑动帧率。
- **动态页眉**: 导航栏会根据当前深度自动切换显示“返回”按钮或“角色 ID”标识。

# 项目愿景与 UI 设计提案 (Project Akasha)

## 1. 视觉语言定义 (Visual Language)
Project Akasha 旨在通过“精密、深邃、通透”的视觉风格，将原本枯燥的伤害计算提升为一种“数字复盘仪式”。

### 1.1 核心材质：星云玻璃 (Nebula Glass)
*   **底色**: 采用深色径向渐变 (#0F0E17 -> #1A1926)，模拟深空环境。
*   **磨砂**: 使用 `ft.Blur(30, 30)`，背景不透明度 `0.4`。
*   **边缘**: 双层纳米边框。外层 1.5px 半透明主色，内层 0.5px 高亮。
*   **光晕**: 放弃阴影，改用与主色一致的长距离低透明度 Glow。

### 1.2 状态色体系 (Functional Palette)
*   **筹备态 (Mutable)**: 淡紫色 (#D1A2FF) - 代表自由与灵活。
*   **仿真态 (Locked)**: 科技蓝 (#4CC2F1) - 代表严谨与结果。
*   **警告态 (Alert)**: 赤金色 (#FFE699) - 代替刺眼的红色。

## 2. 交互主基调 (Interaction Tone)

### 2.1 状态机工作流
1.  **筹备期**: 左右平铺布局，UI 强调“可编辑性”（如微弱的呼吸描边）。
2.  **仿真期**: 进入不可逆状态，界面色调微调。
3.  **审计期**: 界面进入“上帝视角”，数据实时同步。

### 2.2 动画规范
*   **空间平移 (Spatial Slide)**: 画布横向平移，暗示逻辑的线性流转。
*   **数据活化 (Number Roll)**: 数值跳变采用极短的滚动动效，赋予数据“流”的特质。

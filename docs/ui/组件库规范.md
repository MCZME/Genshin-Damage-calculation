# 组件库规范 (V5.0)

## 1. 核心视图组件 (Views)
### `StrategicView` (战略视图)
**位置**: `ui/views/strategic_view.py`
**特性**: 
- **MVVM 绑定**: 绑定 `StrategicPageViewModel`。
- **性能优化**: 采用 **Active Proxy** 模式，通过 `ActiveCharacterProxy` 实现详情面板的实例复用，切换角色零卡顿。

### `SceneView` (场景视图)
**位置**: `ui/views/scene_view.py`
**功能**: 通过 `TargetViewModel` 实现怪物属性与空间坐标的逻辑聚合。

### `TacticalView` (战术视图)
**位置**: `ui/views/tactical_view.py`
**功能**: 通过 `TacticalPageViewModel` 驱动动作序列流，解耦元数据发现逻辑。

## 2. 业务原子组件 (Components)
### `MemberSlot`
**位置**: `ui/components/strategic/member_slot.py`
**用途**: 绑定 `CharacterViewModel`，展示头像、等级、武器及圣遗物摘要。

### `WeaponCard`
**位置**: `ui/components/strategic/weapon_card.py`
**用途**: 绑定 `WeaponViewModel`，集成图标预览与基于 VM 的强类型滑块。

### `ArtifactSlot`
**位置**: `ui/components/strategic/artifact_slot.py`
**用途**: 绑定 `ArtifactPieceViewModel`，彻底解耦数据库访问逻辑。

### `ActionCard`
**位置**: `ui/components/tactical/action_card.py`
**用途**: 绑定 `ActionViewModel`，展示动作名称映射、角色标识及选中光点。

### `AssetGrid`
**位置**: `ui/components/common/asset_grid.py`
**特性**: 接收 `AssetOption` DTO 列表，支持高性能的声明式过滤与搜索。

## 3. 基础设施服务 (Services)
### `MetadataService`
**位置**: `ui/services/metadata_service.py`
**职责**: 集中管理全量资产元数据缓存，提供已实装名单。

### `SimulationService`
**位置**: `ui/services/simulation_service.py`
**职责**: 封装仿真任务生命周期，解耦 `AppState` 与仿真引擎细节。

### `UIFormatter`
**位置**: `ui/services/ui_formatter.py`
**职责**: 静态格式化工具（颜色映射、图标检索、数值千分位）。

---

## 4. 开发建议 (V5.0)
- **MVVM 强制要求**: 严禁在组件内直接读写 `dict`。所有数据交互必须经过 `DataModel` -> `ViewModel` 链路。
- **列表性能**: 角色/武器等长列表必须使用轻量级 `AssetOption` DTO 渲染，避免为非活动项创建重型 VM。
- **稳定性**: 复杂面板必须通过 **Proxy ViewModel** 保持控件树稳定，禁止全量重建。

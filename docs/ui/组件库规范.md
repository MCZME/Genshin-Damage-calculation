# 重构版组件库规范

## 1. 核心视图组件 (Views)
### `StrategicView` (战略视图)
**位置**: `ui/views/strategic_view.py`
**特性**: 
- **全屏响应式**: 自持 180px 侧边编队栏。
- **模板持久化**: 集成 `PersistenceManager`，支持异步保存与加载。
- **性能优化**: 采用 `workbench_cache` 机制，切换成员时实现亚秒级渲染。

### `SceneView` (场景视图)
**位置**: `ui/views/scene_view.py`
**功能**: 定义敌人属性与空间坐标。集成 `SpatialRadar` 可视化雷达。

### `TacticalView` (战术视图)
**位置**: `ui/views/tactical_view.py`
**功能**: 编排动作序列流。支持可视化卡片排序与元数据驱动的参数微调。

## 2. 业务原子组件 (Components)
### `MemberSlot`
**位置**: `ui/components/member_slot.py`
**用途**: 战略视图侧边栏的角色槽位，展示头像、武器及圣遗物摘要。

### `WeaponCard`
**位置**: `ui/components/weapon_card.py`
**用途**: 一体化武器配置卡，集成图标预览与等级/精炼滑块。

### `ArtifactSlot`
**位置**: `ui/components/artifact_slot.py`
**用途**: 圣遗物单个部位的详细编辑器。

### `SpatialRadar`
**位置**: `ui/components/spatial_radar.py`
**用途**: 场景视图中的空间雷达可视化。

### `TargetSidebarSlot`
**位置**: `ui/components/target_sidebar_slot.py`
**用途**: 场景视图侧边栏的目标实体槽位。

### `TacticalActionBtn`
**位置**: `ui/components/tactical_action_btn.py`
**用途**: 战术视图招式指令按钮，自适应元素主题色。

## 3. 功能性组件
### `PropertySlider`
**位置**: `ui/components/property_slider.py`
**特性**: 浏览/编辑双态切换，全局独占式焦点管理。

### `AssetGrid`
**位置**: `ui/components/asset_grid.py`
**用途**: 通用的角色/武器资产选择网格。

## 4. 基础设施服务 (Services)
### `UIEventBus`
**位置**: `ui/services/event_bus.py`
**职责**: 核心事件总线，负责 UI 状态变更的发布与订阅（Observer Pattern）。

### `UIFormatter`
**位置**: `ui/services/ui_formatter.py`
**职责**: 静态格式化工具，集中处理颜色映射、文字截取、数值美化等逻辑。

### `PersistenceManager`
**位置**: `ui/services/persistence_manager.py`
**职责**: 异步持久化服务，接管所有 `FilePicker` 与 JSON 文件 I/O。

---

## 5. 开发建议 (V3.2)
- **解耦逻辑**: 严禁在组件内部编写复杂的业务计算，优先调用 `UIFormatter`。
- **状态通知**: 状态变更后应通过 `app_state.events.notify()` 发送通知，由订阅者（如 Layout 或 View）决定如何刷新。
- **路径规范**: 所有状态类位于 `ui/states/`，所有公共组件位于 `ui/components/`。

# 重构版组件库规范

## 1. 核心视图组件
### `StrategicView` (战略视图)
**位置**: `ui/views/strategic_view.py`
**特性**: 
- **全屏响应式**: 摒弃旧版三栏架构，自持 180px 侧边编队栏。
- **模板持久化**: 集成 `PersistenceManager`，支持单角色模版、圣遗物套装、全量配置的异步保存与加载。
- **性能优化**: 采用 `workbench_cache` 机制，切换成员时实现亚秒级渲染。

### `SceneView` (场景视图)
**位置**: `ui/views/scene_view.py`
**功能**: 定义敌人属性与空间坐标。内置可视化雷达图。

### `TacticalView` (战术视图)
**位置**: `ui/views/tactical_view.py`
**功能**: 编排动作序列流。支持可视化卡片排序与参数微调。

## 2. 核心原子组件
### `PropertySlider` (属性滑块)
**位置**: `ui/reboot/components/property_slider.py`
**特性**: 
- **双态切换**: 浏览态显示文本，点击后原地置换为滑块。
- **调度管理**: 全局独占式焦点管理，同一时间仅能激活一个编辑项。

### `ArtifactSlot` (圣遗物插槽)
**位置**: `ui/reboot/components/artifact_slot.py`
**用途**: 圣遗物单个部位的全量配置入口。
**功能**: 
- **套装映射**: 自动获取数据库/注册表中的圣遗物套装列表。
- **词条录入**: 支持主词条属性+数值、4个副词条的录入。
- **局部刷新**: 更新数据时仅重绘内部控件，不触发整页刷新。

### `StatInputField` (数值输入)
**位置**: `ui/reboot/components/stat_input.py`
**用途**: 用于副词条、抗性等数值录入。
**特性**: 获得焦点时触发对应元素属性色呼吸光晕。

### `AssetGrid` (资产网格)
**位置**: `ui/reboot/components/asset_grid.py`
**用途**: 用于角色、武器选择的选择器界面。

## 3. 持久化服务
### `PersistenceManager`
**位置**: `ui/services/persistence_manager.py`
**职责**: 处理所有文件 I/O 与 `FilePicker` 交互。
**模式**: 异步返回路径 (Async Return Path)，解耦 UI 与文件系统。

---

## 4. 开发建议 (V3.1)
- **精准 update()**: 优先修改子控件属性并调用 `control.update()`，而非重构整个 `controls` 列表。
- **缓存优先**: 对于复杂的面板（如角色工作台），在 View 层级维护 `dict` 缓存已生成的控件树。
- **注入服务**: 复杂逻辑（如保存文件）应在 `PersistenceManager` 中定义，通过 `page.persistence` 注入 View 调用。

# UI 设计：批处理宇宙视图 (V4.0)

## 1. 设计概览

批处理宇宙采用 **"三视图路由架构"**，通过 `UniverseRouter` 管理编辑器、运行、分析三个独立视图，每个视图拥有独立状态层，由 `BatchUniverseState` 统一协调。

```
┌─────────────────────────────────────────────────────────────────┐
│                     BatchUniverseState                          │
│  ┌──────────────┬──────────────┬──────────────┐                │
│  │EditorState   │ RunState     │AnalysisState │                │
│  └──────┬───────┴──────┬───────┴──────┬───────┘                │
│         │              │              │                         │
│  ┌──────▼──────┐┌──────▼──────┐┌──────▼──────┐                 │
│  │UniverseView ││RunView      ││AnalysisView │                 │
│  │  编辑器      ││  运行       ││  分析        │                 │
│  └─────────────┘└─────────────┘└─────────────┘                 │
│         │              │              │                         │
│         └──────────────┴──────────────┘                         │
│                   UniverseRouter                                 │
│                   / → /run → /analysis                          │
└─────────────────────────────────────────────────────────────────┘
```

class BatchUniverseState:
    """分支宇宙子应用的状态协调器。"""

    def __init__(self) -> None:
        self.editor_state = BatchEditorState()
        self.run_state = BatchRunState()
        self.analysis_state = BatchAnalysisState()

- 持有三个子状态的引用
- 提供 `attach_page()`、`attach_branch_queue()` 连接基础设施

### 3.3 布局算法

采用 **自底向上测量 + 自顶向下放置** 的布局策略：

```python
def measure(node: BatchNode) -> int:
    """计算节点跨度（子树覆盖的叶子数）。"""
    if not node.children:
        return 1  # 叶子节点跨度为1
    return max(1, sum(measure(child) for child in node.children))

def place(node: BatchNode, depth: int, leaf_offset: int) -> None:
    """根据跨度计算节点居中位置。"""
    span = spans[node.id]
    center_index = leaf_offset + (span - 1) / 2  # 居中计算
    positions[node.id] = (
        base_x + depth * x_gap,
        base_y + center_index * y_gap,
    )
```

### 6.2 UniverseRouter 组件

位置：`ui/views/universe_router.py`

```python
@ft.component
def UniverseRouter(
    universe_state: BatchUniverseState,
    editor_state: BatchEditorState,
    run_state: BatchRunState,
    analysis_state: BatchAnalysisState,
):
    router, _ = ft.use_state(UniverseRouterState(route=...))

    ft.context.page.on_route_change = router.route_change
    ft.context.page.on_view_pop = router.view_popped

    return build_universe_views(router.route, universe_state)
```

### 6.3 视图栈构建

```python
def build_universe_route_stack(route: str | None) -> list[str]:
    """构建声明式视图栈。"""
    resolved = resolve_universe_route(route)
    if resolved == UNIVERSE_EDITOR_ROUTE:
        return [UNIVERSE_EDITOR_ROUTE]
    return [UNIVERSE_EDITOR_ROUTE, resolved]  # 编辑器始终在栈底

def build_universe_views(route: str, universe_state) -> list[ft.View]:
    """根据路由栈构建 Flet 视图实例。"""
    views = []
    for route_item in build_universe_route_stack(route):
        if route_item == UNIVERSE_EDITOR_ROUTE:
            views.append(UniverseView(...))
        elif route_item == UNIVERSE_RUN_ROUTE:
            views.append(UniverseRunView(...))
        elif route_item == UNIVERSE_ANALYSIS_ROUTE:
            views.append(UniverseAnalysisView(...))
    return views
```

## 7. 独立进程架构

### 7.1 进程入口

位置：`ui/universe_launcher.py`

```python
def start_universe_process(
    page: ft.Page,
    branch_to_main_queue: mp.Queue,
    initial_message: dict,
):
    """批处理编辑器独立进程入口。"""
    state = BatchUniverseState()
    state.attach_page(page)
    state.attach_branch_queue(branch_to_main_queue)

    # 处理 INIT_BATCH_EDITOR 初始化消息
    if initial_message.get("type") == "INIT_BATCH_EDITOR":
        state.initialize_project(
            initial_message["base_config"],
            initial_message.get("project_name", "批处理项目")
        )

    # 启动消息监听循环
    _start_message_listener(state, branch_to_main_queue)

    # 渲染 UI
    page.add(UniverseRouter(state, ...))
```

### 7.2 IPC 通信消息类型

| 常量 | 方向 | 含义 |
|------|------|------|
| `INIT_BATCH_EDITOR` | Main → Branch | 初始化编辑器 |
| `BRANCH_RUN_BATCH_REQUEST` | Branch → Main | 提交批处理执行请求 |
| `MAIN_BATCH_PROGRESS` | Main → Branch | 执行进度更新 |
| `MAIN_BATCH_TASK_RESULT` | Main → Branch | 单任务结果 |
| `MAIN_BATCH_FINISHED` | Main → Branch | 执行完成 |
| `MAIN_BATCH_REJECTED` | Main → Branch | 执行被拒绝 |

## 8. 交互规范

### 8.1 视图切换

- 三个视图顶部均显示导航按钮
- 当前视图按钮禁用，其他可点击
- 使用 `page.push_route()` 进行路由跳转

### 8.2 运行视图状态机

```
等待执行 → 编译中 → 已提交 → 执行中 → 完成
                ↓                    ↓
              编译失败             部分失败
```

### 8.3 操作约束

| 状态 | 停止按钮 | 查看分析按钮 | 编辑操作 |
|------|---------|-------------|---------|
| 等待执行 | 禁用 | 禁用 | 允许 |
| 执行中 | 启用 | 禁用 | 禁用 |
| 完成 | 禁用 | 启用 | 允许 |

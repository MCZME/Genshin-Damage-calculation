# 角色面板磁贴组件说明 (V6.8 Pro - Snapshot Version)

> 本文档定义了分析视图中用于展示角色瞬时属性快照及生存状态的磁贴组件 `CharacterStatsTile` 的设计与交互规范。

---

## 1. 组件定位 (Positioning)

*   **核心职能**：展示特定仿真帧（Frame N）下，指定角色的完整属性切片与生存状态。
*   **纯粹瞬时态**：组件仅关注“当前瞬间”的数值，不包含时序曲线。
*   **数据驱动**：基于数据库离散记录（Jumps & Lifecycles）实时还原瞬时状态。
*   **网格规格**：**2x2** (标准单元格单元)。

---

## 2. 正常态：仪表盘模式 (Normal State)

### 2.1 交互逻辑 (Action Cluster)
*   **设置按钮 (Settings)**：**切换焦点角色 (Switch Character)**。
    *   点击后弹出菜单，列出当前 Session 中的所有角色。
    *   切换后，磁贴将立即刷新为新角色的数据。
*   **展开按钮 (Expand)**：**开启详情配置模式 (Expanded Detail)**。

### 2.2 展示内容
*   **Identity Bar (标题栏)**：
    *   **左侧**：元素图标 + 角色名称。
    *   **下方**：**生存状态条组 (StatusBar)**。
        *   **HP 条 (绿色)**：当前血量 / 最大血量 (包含实时加成)。
        *   **能量条 (元素色)**：当前能量 / 爆发所需能量。
    *   **右侧**：当前帧编号（如 `F_600`）。
*   **Hero Area (数值网格)**：
    *   **布局**：2x4 网格。
    *   **内容**：展示在“展开态”中被用户**勾选 (Checked)** 的属性。
    *   **视觉**：最终值（W800）+ 绿字加成值（GREEN_400）。
*   **Status Footer (状态页脚)**：
    *   **左侧：活跃效果墙**。以胶囊标签显示所有非数值效果（如“抗打断”、“夜魂”）。
    *   **护盾汇总**：若存在护盾，显示带有 `SHIELD` 图标的标签，数值为所有活跃护盾实例的剩余量之和。

---

## 3. 展开态：审计与配置模式 (Expanded State)

### 3.1 左侧：属性配置列表 (Checkable List)
*   **Checkbox 机制**：用户勾选哪些属性，关闭展开态后，正常态的磁贴网格中就只显示这些属性（上限 8 个）。
*   **独立偏好**：勾选配置基于角色 ID 独立保存。

### 3.2 右侧：属性审计详情 (Audit & Logic)
*   **计算公式还原**：顶部以数学公式形式展示该属性的最终合成路径。
    *   例：`攻击力 = 基础值 × (1 + 攻击力%) + 固定值`
*   **审计堆栈 (The Stack)**：
    *   逐行列出当前帧所有对该属性产生贡献的 **Modifier (修饰符)**。
    *   **展示项**：`[图标] 来源名称 : 贡献值 (运算类型)`。

---

## 4. 技术契约 (Technical Contract)

### 4.1 状态还原机制 (State Reconstruction)
*   **HP/能量**：通过查询 `simulation_state_jumps` 表中 `frame_id <= T` 的最新一条记录还原。
*   **面板属性**：结合 `char_base` (初始快照) 与 `modifier_lifecycles` (覆盖 T 帧的活跃区间) 实时计算。
*   **护盾/效果**：从 `simulation_effect_lifecycles` 过滤出活跃实例，若为护盾则需进一步查询 `simulation_shield_jumps`。

### 4.2 元数据集成
*   **元素识别**：磁贴通过 `AppState.char_map` 根据名称反查角色元素类型，以应用正确的 `theme_color` 与 `Icon`。

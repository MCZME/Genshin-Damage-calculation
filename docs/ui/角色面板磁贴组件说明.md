# 角色面板磁贴组件说明 (V9.10 MVVM - Pro)

> 本文档定义了分析视图中用于展示角色瞬时属性快照及生存状态的磁贴组件 `CharacterStatsTile` 的设计与交互规范。

---

## 1. 组件定位 (Positioning)

*   **核心职能**：展示特定仿真帧（Frame N）下，指定角色的完整属性切片、生存状态及活跃效果。
*   **架构模式 (V9.10)**：
    *   **结构化审计**：引入 `AuditResult` 与 `ZonedModifier` 协议，将计算逻辑从纯字符串解析进化为**语义化数据驱动**。
    *   **混合缓存机制**：具备快照引用缓存、计算值缓存及分解 (Breakdown) 缓存，支持高频帧切换下的零延迟渲染。
    *   **分层渲染架构**：由 `StatsViewModel` 驱动，下接 `InlineFormulaChain` 与 `ModifierCardGrid` 等原子组件。
*   **组件规格**：**2x2** (仪表盘态) / **全屏** (展开审计态)。

---

## 2. 正常态：仪表盘模式 (Normal State)

### 2.1 布局结构
*   **顶部标题行 (Top Row)**：
    *   **元素图标 + 角色名称**。
    *   **右侧**：当前帧号 (例: `F_1024`)，使用 Consolas 字体。
*   **状态栏区域 (Status Bar Area)**：
    *   **自适应状态集群 (AdaptiveStatusCluster)**：动态渲染 HP 与能量条。支持点击选中特定资源进入底层审计。
*   **属性网格 (Stat Grid)**：2xN 的数值布局，显示最终值及来自修饰符的增益部分（绿色 `+N`）。
*   **底部状态墙 (Status Capsule Wall)**：
    *   **状态胶囊**：显示 2 字符截断的微型胶囊，琥珀色背景。

---

## 3. 展开态：三层交互架构 (V9.10 Triple-Layer Architecture)

### 3.1 顶部：整合全景看板 (Integrated Header)
*   **元素感知设计**：背景与图标根据角色元素动态变色。
*   **状态实时联动**：展示大尺寸水位条，点击任意水位（如血条）可直接在下方审计面板切换至“生存状态审计”模式。

### 3.2 中部：二分监控矩阵 (Split Monitor Matrix)
*   **左区 (70%) - 属性配置矩阵 (V9.5 Pro V2)**：
    *   **矩阵平衡布局**：基础属性、进阶属性、元素加成（上/下）分块平铺。
    *   **实时预览**：每个属性勾选项右侧实时显示当前帧的计算结果。
*   **右区 (30%) - 活跃效果墙**：
    *   **StatusCapsuleGrid**：展示所有活跃效果的药丸型图标，带剩余帧数倒计时与进度条。

### 3.3 底部：结构化审计中心 (Structured Audit Center - V9.9+)
*   **内联式公式链 (Inline Formula Chain - V9.8)**：
    *   **融合视觉**：修饰符卡片（ModifierInlineCard）直接嵌入公式符号（+、×）之间。
    *   **乘区色彩编码**：
        *   **琥珀色 (Base)**：基础白值区。
        *   **绿色 (Percent)**：百分比加成区（累乘型专用）。
        *   **青蓝色 (Flat)**：固定值/面板累加区。
    *   **交互高亮**：点击公式中的特定域（Domain），下方修饰符网格将自动高亮对应乘区的贡献来源。
*   **来源分组修饰符网格 (Source-Grouped Grid - V9.9)**：
    *   **智能分类**：根据修饰符元数据自动归类为：**武器 (Weapon)**、**圣遗物 (Artifact)**、**天赋技能 (Talent)**、**共鸣 (Resonance)** 及 **其他 (Other)**。
    *   **细节感知**：每张卡片展示来源简名及具体贡献数值，支持负值（红色）显示。

---

## 4. 技术契约 (Technical Contract)

### 4.1 审计引擎协议 (V9.7)
*   **计算范式 (Paradigm)**：支持 `cumulative` (累乘)、`pct_additive` (百分比累加) 及 `additive` (纯累加)。
*   **元素自动合并**：计算“伤害加成”时，系统自动扫描并合并角色对应元素的特定加成项。

### 4.2 效果描述系统 (V9.6)
*   **动态注入**：支持在效果描述中使用 `{atk}` 等占位符，由 VM 实时从 `metrics` 快照中提取数据并进行模板化渲染。

### 4.3 性能保障 (Performance)
*   **竞态保护**：`fetch_snapshot` 具备请求版本控制，防止在网络/数据库波动时旧数据覆盖新帧。
*   **异步健壮性 (V9.10)**：修复了异步抓取期间的缓存竞态问题，确保在快照加载完成前不缓存临时的空结果。

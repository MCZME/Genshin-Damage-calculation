# 角色面板磁贴组件说明 (V9.6 MVVM - Pro)

> 本文档定义了分析视图中用于展示角色瞬时属性快照及生存状态的磁贴组件 `CharacterStatsTile` 的设计与交互规范。

---

## 1. 组件定位 (Positioning)

*   **核心职能**：展示特定仿真帧（Frame N）下，指定角色的完整属性切片、生存状态及活跃效果。
*   **架构模式 (V9.6)**：
    *   **统一渲染入口**：使用 `CharacterStatsTileRenderer` 统一管理 `StatsViewModel` 的生命周期，确保数据一致性。
    *   **纯 UI 解耦**：`StatsDashboard` 与 `StatsDetailAudit` 进化为纯渲染组件，不再持有内部状态。
    *   **数据驱动**：由 `StatsViewModel` 负责所有快照抓取、公式计算与审计链追踪。
*   **组件规格**：**2x2** (仪表盘态) / **全屏** (展开审计态)。

---

## 2. 正常态：仪表盘模式 (Normal State)

### 2.1 布局结构
*   **顶部标题行 (Top Row)**：
    *   **元素图标 + 角色名称**。
    *   **右侧**：当前帧号 (例: `F_1024`)，使用 Consolas 字体增强科技感。
*   **状态栏区域 (Status Bar Area)**：
    *   **紧凑型状态条 (CompactStatusBar)**：支持多条资源（生命值、能量等）的并排显示。
    *   **动态性**：仅显示用户配置为“可见”的资源项。
*   **属性网格 (Stat Grid)**：2xN 的数值布局，显示最终值及来自修饰符的增益部分（绿色 `+N`）。
*   **底部状态墙 (Status Capsule Wall)**：
    *   **状态胶囊**：显示前 2 个字符的微型胶囊，琥珀色背景，悬浮显示完整名称、描述及剩余帧数。
    *   **护盾标识**：独立显示的护盾 HP 总量图标。

### 2.2 交互逻辑
*   **设置按钮 (Settings)**：切换当前磁贴监控的焦点角色。
*   **展开按钮 (Expand)**：进入全屏审计模式。

---

## 3. 展开态：二分监控矩阵 (V9.6 Split Monitor Matrix)

### 3.1 顶部：整合全景看板 (Integrated Header)
*   **自适应状态集群 (AdaptiveStatusCluster)**：使用响应式布局展示大尺寸资源条（HP/能量），支持点击切换审计焦点。
*   **背景复盘**：在审计下方细节时，顶部始终固定显示该帧的生存底色。

### 3.2 中部：二分功能区
*   **左区 (70%) - 属性配置矩阵**：
    *   **矩阵结构重构 (V9.5 Pro V2)**：
        *   **基础属性**：生命、攻击、防御、精通。
        *   **进阶属性**：双暴、充能、治疗加成、护盾强效。
        *   **元素加成 (拆分)**：由于属性项较多，拆分为“元素加成·上/下”两组以实现矩阵视觉均衡。
    *   **横向平铺**：按分类平铺复选框，每个选项右侧实时显示当前帧的计算结果。
*   **右区 (30%) - 活跃效果墙**：
    *   **状态胶囊网格 (Expanded Mode)**：展示所有活跃效果的药丸型图标。
    *   **可视化进度**：每个胶囊自带剩余帧数显示及百分比进度条（由右向左消耗）。

### 3.3 下方：动态审计详情层 (Audit Panel)
*   **智能审计引擎 (V9.3+)**：
    *   **公式解析**：展示所选属性的原始合成公式（例：`Base * (1 + %) + Flat`）。
    *   **元素自动合并**：计算“伤害加成”时，系统会自动识别角色元素并累加基础属性中的对应元素伤害加成，实现无缝审计。
*   **修饰符堆栈**：横向滚动展示所有贡献该属性的修饰符卡片（来源名 + 贡献数值）。
*   **资源详情**：若选中血条/能量，自动切换至大字体水位展示，不显示公式。

---

## 4. 技术契约 (Technical Contract)

### 4.1 状态胶囊系统 (Status Capsule System)
*   **Mini 模式**：2 字符截断，用于高密度仪表盘。
*   **Expanded 模式**：全名 + 进度条，用于审计详情。
*   **描述支持**：V9.6 新增 `description` 字段，支持在 Tooltip 中展示效果的详细说明。

### 4.2 数据流与缓存
*   **单一 ViewModel**：由 Renderer 管理单一实例，避免多层组件重复创建带来的内存开销。
*   **异步快照**：具备版本保护机制，确保 UI 渲染与 `frame_id` 严格对齐，防止数据错位。

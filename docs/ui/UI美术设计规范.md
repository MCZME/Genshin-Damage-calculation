# UI 美术设计规范 V3.1 (Atomic Refinement)

## 1. 核心配色与材质
*(保持 V3.0 基准：Astral Deep Purple)*
*   **背景**: `#1A1625`
*   **卡片**: `#252131` (SURFACE)
*   **交互高亮**: `#D1A2FF` (雷光紫)

## 2. 沉浸式组件设计

### 2.1 Hero Header (沉浸式页眉)
用于详情编辑器顶部：
*   **标题**: 24px W_900 暖白文字。
*   **副标题**: 10px Bold，使用该角色的元素强调色。
*   **下边框**: 1px solid `rgba(255,255,255,0.05)`。
*   **操作按钮**: 右侧放置“保存模板”等原子化操作。

### 2.2 Dashboard Overview (配置仪表盘)
用于总览卡片：
*   **两列对齐**: 左侧强调名字，右侧强调属性摘要（等级、天赋、命座）。
*   **属性标签**: 小字号 Bold 标签，背景使用 `rgba(255,255,255,0.05)`，支持自动换行。

### 2.3 选人卡片 (Picker Card)
*   **比例**: `0.72` 纵向比例。
*   **视觉锚点**: 中心显示 36px 图标，底层叠加 40px 元素文字水印。
*   **状态标记**: 未实现项通过 `Lock` 图标和 `0.4` 透明度灰度处理。

## 3. 原子化保存 UI 规范
*   **操作按钮**: 采用 `ft.TextButton` 配合 `UPLOAD_FILE` / `DOWNLOAD` 图标。
*   **弹窗列表**: 采用 `ft.ListTile` 展示 JSON 文件名，保持简洁。

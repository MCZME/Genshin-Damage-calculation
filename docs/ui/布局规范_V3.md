# UI 布局规范 (V3 - 一体化看板版)

## 1. 全局架构 (AppLayout)
采用 **“顶部导航 + 弹性主体 + 底部状态栏”** 的经典结构。

### 1.1 空间分配 (Spatial Distribution)
- **Header**: 宽度填满，固定高度，包含全局标题、导航胶囊及配置保存/读取按钮。
- **Body**: 核心内容区。在主流视图中，`AppLayout` 的原始左右侧边栏会被隐藏，由各个视图内部自行管理局部侧边栏。
- **Footer**: 固定高度 `50px`，包含仿真进度条、批处理入口及“开始仿真”大按钮。

### 1.2 顶部导航 (Header Navigation)
- 导航胶囊 (NavItems) 在激活态底部增加 **`3px` 主题色指示线**（带高度伸缩动画），并同步放大字体及图标，提升视觉反馈。

## 2. 视图内局部布局 (Local Layout)
战略、场景、战术视图均遵循 **“局部侧边栏 + 动态工作台”** 的双栏结构。

- **局部侧边栏**: 宽度固定 `180px`。
  - 战略视图：显示编队 4 人槽位。
  - 场景视图：显示怪物目标列表。
  - 战术视图：显示角色选择器。
- **动态工作台**: 弹性宽度 `expand=True`。
  - 内部通常分为“上半区（核心概览）”与“下半区（详细配置/圣遗物/序列轴）”。

## 3. 响应式策略 (Responsive Strategy)
- **断点 (Breakpoint)**: `1400px` (侧边栏模式切换) 与 `1000px` (视觉简化)。
- **尺寸锁定原则**: 组件在“浏览态”与“编辑态”下应尽可能保持外部尺寸一致，避免“布局抖动”。
- **滚动支持**: 所有看板内部均支持 `ft.ScrollMode.ADAPTIVE`，防止纵向溢出。

## 4. 视觉层级与材质
- **背景**: 深色背景 (`#1A1625`)。
- **玻璃拟态**: Header 和 Footer 采用 `ft.Blur(20, 20)` 增加通透感。
- **边框**: 统一使用 `rgba(255, 255, 255, 0.05)` 作为基础分割线。

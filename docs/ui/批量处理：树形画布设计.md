# UI 设计：批处理编辑器 (V3.3)

## 1. 设计核心 (Concept)
采用 **“集中控制 + 自动展开”** 模式。通过可视化画布构建仿真逻辑，并将复杂的路径配置收拢于高度集成的右侧检视器中。

## 2. 交互组件规范

### 2.1 HUD (状态监控)
*   **模拟样本**: 实时统计当前树的 **叶子节点** 数量。该数值代表点击“执行批量仿真”后实际运行的任务数。
*   **持久化**: 新增磁盘与文件夹图标，支持方案的独立存取。

### 2.2 UniverseCanvas (核心画布)
*   **叶子节点**: 右下角显示 **“SAMPLE”** 标签，明确标识计算终点。
*   **受控节点**: 黄色调高亮，禁用单独删除逻辑。
*   **背景点击**: 全屏手势拦截，点击空白处自动取消选中并收起检视器。

### 2.3 MutationInspector (检视器)
*   **节点编辑**: 标题处支持实时修改节点名称。
*   **区间模式**: 内置开关，支持一键将普通节点转化为“区间锚点”。
*   **逻辑锁**: 已有子节点的节点（锚点）禁止添加自身规则，确保变异路径不发生冲突。

## 3. 视觉风格 (UI/UX)
*   **玻璃拟态**: 检视器背景调优为 `rgba(80, 70, 100, 0.6)`，配合 2px 发光边框，提升专业感。
*   **连线**: 明紫色 `#D1A2FF` 带箭头，指引变异演化方向。

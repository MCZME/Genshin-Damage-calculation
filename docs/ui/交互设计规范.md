# UI 交互设计规范

## 1. 核心交互流 (Primary User Journey)

### 第一步：战略准备 (Strategic Preparation)
**目标**: 用户通过装配静态资产，确立战斗的“基础变量”。
1. **角色选定**:
   - **交互**: 点击左侧 Slot -> 弹出 AssetGrid。
   - **反馈**: 选中角色后，左侧 Slot 产生元素色呼吸灯；中央看板自动同步为该角色数据。
2. **资产参数化**:
   - **交互**: 在看板中调整等级、命座、天赋。
   - **规范**: 采用 `PropertySlider` 浏览/编辑双态切换，减少空间占用。
3. **圣遗物录入**: 
   - **组件**: 使用 `ArtifactSlot` (五轴并列)，从左向右依次检查花、羽、沙、杯、头。
   - **操作**: 下拉选择套装，手动录入主副词条数值。
4. **模版操作**:
   - **交互**: 点击角色头部的 [保存/读取] 图标。
   - **功能**: 支持导出单个角色的全量配置（含武器、圣遗物）为 `.json` 模版，并在其他槽位快速复用。

### 第二步：场景配置 (Scene Composition)
**目标**: 用户通过定义敌人属性与空间分布，确立战斗的“外部约束”。
1. **实体编排**:
   - **交互**: 在左侧目标侧边栏进行实体切换或增删。
   - **反馈**: 选中目标呈现红色呼吸灯，中央工作台数据同步刷新。
2. **空间定位**:
   - **交互**: 通过雷达图可视化所有目标位置，通过 TextField 精确录入 X/Z 坐标。
3. **抗性审查**:
   - **交互**: 在下半区网格中快速输入各元素抗性数值。

### 第三步：战术编排 (Tactical Sequencing)
**目标**: 用户通过定义动作序列，确立战斗的“动态意图”。
1. **指令录入**: 点击角色技能按钮，动作卡片即追加至右侧序列轴。
2. **序列调整**: 支持点击选中卡片进行参数微调，或通过 [左/右] 移动按钮调整动作先后顺序。

## 2. 状态感知交互 (State-Aware Interaction)

### 2.1 浏览态 vs 编辑态 (Browse vs. Edit Mode)
为了解决信息密度与交互空间的冲突，核心控制器（如滑块）支持双态切换：
- **浏览态 (Default)**: 仅显示属性名与当前值，占用空间极小。
- **编辑态 (Active)**: 点击模块后，原地平滑展开滑块控制器。完成编辑后点击外部区域自动收回。

### 2.2 独占焦点原则
界面同一时间仅允许一个组件处于“编辑态”，确保用户的注意力聚焦，并防止布局逻辑混乱。

### 3.3 分析磁贴交互 (V4.5 Pro)
磁贴操作遵循 **“按需显现 (Progressive Disclosure)”** 原则：
- **悬浮态 (Hover)**: 
    - 鼠标进入磁贴区域时，右上角平滑渐现 **操作集群 (Action Cluster)**。
    - 磁贴边框与背景同步高亮，提供明确的焦点反馈。
- **操作闭环**:
    - **设置 (Settings)**: 呼出侧边抽屉或弹出层，进行局部数据切片配置。
    - **展开 (Maximize)**: 触发全屏聚焦模式，屏蔽背景干扰。
    - **关闭 (Close)**: 移除磁贴实例，同步清理数据订阅，属于不可逆操作。

## 4. 响应式布局准则

### 3.1 空间优先级 (Spatial Priority)
- **侧边导航**: 固定宽度 `180px`，用于承载资产列表或实体列表。
- **中央看板**: 弹性宽度，用于承载核心配置逻辑。
- **全局边距**: `15px`，确保在不同分辨率下均有呼吸感。

### 3.2 镜像视觉原则 (Mirror Visual Principle)
不同视图（战略/场景/战术）在架构上复刻“侧边栏 + 工作台”结构，降低用户的认知切换负荷。

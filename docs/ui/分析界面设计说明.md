# 分析界面设计说明 (V4.0 - 复盘系统)

> 本文档定义了仿真引擎 V4.0 下复盘系统的核心功能架构、数据流转逻辑以及交互层级规范。

---

## 1. 架构理念 (System Architecture)

### 1.1 分层叠加模型 (Layered Overlay)
分析界面采用三层垂直叠加架构，确保交互的高效与稳定：
1.  **底层 (Layout Row)**: 工具箱 (`AnalysisToolbox`) + 磁贴网格 (`Workspace`)。
2.  **中层 (Floating Drawer)**: 详情抽屉，用于不打断主工作流的深度审计。
3.  **顶层 (Focus Layer)**: 全屏遮罩，用于磁贴最大化及模态组件（如 `HistoryDialog`）。

### 1.2 磁贴标准化协议 (AnalysisTile Protocol)
所有分析模块必须遵循统一接口：
*   `load_data(adapter)`: 负责从离散数据库拉取历史记录并进行片段化转换。
*   `sync_to_frame(frame_id)`: 全局同步回调，仅修改视觉属性，严禁重构 UI 树。
*   `expand 属性`: 控制磁贴在栅格布局中是自适应高度（False）还是填满剩余空间（True）。

---

## 2. 核心功能逻辑

### 2.1 全局时间标尺 (Global Scrubber) —— 全时空同步
*   **特性**: 界面常驻，横跨主工作区底部。
*   **数据驱动**: 启动时预设 0 帧，仿真结束后通过 `update_range` 动态更新最大帧数。
*   **联动效应**: 拖动时向所有激活磁贴广播 `frame_id`，实现曲线、时间轴、物理位置的逐帧联动。

### 2.2 异步审计链路 (Asynchronous Drill-down)
*   **链路**: 点击 DPS 曲线点 -> 跳转帧 -> 唤起 `FloatingDrawer`。
*   **加载机制**: 采用“先占位、后填充”模式。抽屉立即滑出并显示 `AuditPanel` 加载态，后台抓取数据库后通过 `audit_detail_ready` 事件触发局部刷新。

### 2.3 历史记录浏览器 (History Browser)
*   **组件化**: 采用独立 `HistoryDialog` 组件，封装 Artifact Prime 视觉风格。
*   **交互逻辑**: 异步查询 `simulation_sessions` 表，以 `ListTile` 形式展示历史记录，支持一键热加载。

---

## 3. 布局规范 (Layout Constraints)

### 3.1 响应式网格
*   **栅格协议**: 12 列系统。
*   **典型分配**:
    *   **时间轴**: 12/12 (全宽)
    *   **DPS 曲线**: 8/12 或 6/12
    *   **战报看板**: 4/12 (固定高度)

### 3.2 垂直高度管理
*   **锁定策略**: 为了防止网格自动拉伸导致视觉空洞，`TimelineTile` 与 `SummaryTile` 必须显式设置其 `height` 属性，并声明 `expand=False`。

# 原神伤害计算器 - 分析界面设计说明 (V9.2 MVVM)

> 本文档定义了分析系统（Review System）的 V9.1/V9.2 MVVM 架构规范、响应式状态追踪机制及工业级网格布局逻辑。

---

## 1. 架构理念 (System Architecture)

### 1.1 V9.2 响应式追踪模式 (Reactive Tracking Mode)
自 V9.2 起，分析系统引入了更严谨的响应式绑定模式，旨在解决复杂 Flet 组件树下的重绘丢失问题：

1.  **表现层 (AnalysisView)**: 
    - 采用 **“State 注入”** 模式：`build(self, state: AnalysisState)` 方法直接接收 State 实例。
    - **追踪链路**: View 通过 `state.vm` 访问状态属性，这种显式访问路径让 Flet 的渲染引擎能够精准建立属性与 UI 的订阅关系。
2.  **State 代理与协调 (State Coordination)**:
    - `AnalysisState` 不再仅仅是 Model 容器，它作为 **视图控制器的外层接口**，处理跨组件的副作用协调（如：设置全局帧 -> 触发数据订阅 -> 弹出侧边栏）。
    - 外部组件（如 `AnalysisToolbox` 或 `GlobalScrubber`）统一调用 `state` 方法，而非直接读写 `vm`。
3.  **视图模型层 (AnalysisViewModel)**:
    - 依然是纯粹的业务逻辑与布局计算中心。
    - 通过 `@ft.observable` 装饰，持有磁贴列表及布局缓存。

### 1.2 模块化网格布局 (Modular Grid V9.0)
-   **布局容器**: 由 `AnalysisViewModel._calculate_grid_layout` 动态计算。
-   **原子单元 (CELL_SIZE)**: `160px`。
-   **自适应策略**: 
    - 监听窗口 Resize，通过 `state._trigger_rerender()` 强制刷新控件树以适配新宽度。
    - 采用平滑位移动画（`EASE_OUT_EXPO`, 600ms）提升视觉流畅度。

---

## 2. 组件生命周期规范 (V9.2 Hooks)

### 2.1 事件订阅管理
- **全局 PubSub**: 在 `use_effect` 中定义 `setup_events`，监听 `on_force_refresh` 并在清理函数中显式调用 `unsubscribe()`，防止内存泄漏。
- **数据在线确保 (Data Online Assurance)**: 使用 `use_effect` 在组件挂载阶段通过 `DataService.subscribe` 预加载核心元数据（如 `char_base`）。

### 2.2 聚焦层管理 (Focus Layer)
- 系统使用单一 `focus_content` 变量管理模态对话框（`HistoryDialog`）与全屏磁贴模式。
- 点击背景遮罩层（Overlay）时，通过 `state` 统一重置 UI 状态。

---

## 3. 核心交互流程 (Core Interaction)

### 3.1 会话加载流水线
1. 调用 `state.load_session(sid)`。
2. State 协调 ViewModel 标记 `loading = True`。
3. `AnalysisDataService` 批量使旧槽位数据失效 (`invalidate_all_slots`)。
4. 异步 Fetch 完成后，State 驱动 UI 全局更新。

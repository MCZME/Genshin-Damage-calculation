# 原神伤害计算器 - 分析界面设计说明 (V6.8 Pro)

> 本文档定义了复盘系统的核心架构、模块化布局逻辑以及声明式生命周期管理规范。

---

## 1. 架构理念 (System Architecture)

### 1.1 业务驱动生命周期 (Business-Driven Lifecycle)
系统由 **业务状态 (AnalysisState) 直接管控**，遵循 **UI = f(state)** 声明式范式：
1.  **状态即真理**: `AnalysisState` 持有 `active_tiles` 列表及每个磁贴的 `grid_size`。
2.  **Hooks 驱动**: 全面采用 Flet V3 Hooks (`ft.use_state`, `ft.use_effect`) 管理局部 UI 状态。
3.  **零命令式操作**: 严禁在组件内部手动调用 `self.update()`，UI 同步完全由框架渲染引擎接管。

### 1.2 模块化网格布局 (Modular Grid V6.8)
自 V6.8 起，网格系统进行了工业级固化，采用更紧凑的原子单元：
*   **原子单元 (CELL_SIZE)**: `160px` (此前为 220px)。
*   **网格间距 (GUTTER)**: `16px`。
*   **布局引擎**: `calculate_grid_layout` 纯函数算法。采用贪心搜索寻找空槽位并确保像素级对齐。
*   **物理宽度计算**: `phys_w = (units * 160) + ((units - 1) * 16)`。

## 2. 磁贴视觉规范 (V4.5 Pro Standard)

### 2.1 三层空间模型 (Three-Layer Model)
所有分析磁贴必须遵循标准的三层垂直布局，以确保视觉节奏的一致性：
1.  **Identity Bar (顶层)**: 
    *   **呼吸指示灯**: 与内容主题色同步的小圆点，暗示实时数据流。
    *   **动作集群**: 悬浮触发的微型按钮组（设置、展开、关闭）。
2.  **Hero Area (中层)**:
    *   核心展示区。展示 W900 字重的大号数值或全幅脉冲 Canvas 图表。
3.  **Meta Footer (底层)**:
    *   状态标签（如 LIVE_FEED）与实例唯一 ID。

### 2.2 色彩呼应 (Color Echoing)
系统通过 `theme_color` 自动生成磁贴的视觉 DNA：
*   **动态边框**: 边框颜色为主题色的高亮 Tint。
*   **发光渐变**: 背景起始色由主题色的极暗调演变而来。

---

## 3. 布局契约 (Layout Specs)

### 3.1 标准规格 (Grid Size)
*   **全局战报 (SummaryTile)**: **2x1** (宽型看板，V6.8 扩容)。
*   **DPS 趋势图 (DPSChartTile)**: **2x2** (标准正方形图表)。
*   **伤害分布脉冲 (DamageDistTile)**: **4x2** (沉浸式全景布局)。

### 3.2 渲染质量 (Anti-Aliasing)
*   **1.5px 边框**: 解决 Flet/Flutter 引擎在细边缘下的抗锯齿问题。
*   **柔化投影**: 使用扩散半径为 `-2px` 的阴影作为视觉缓冲区，消除圆角衔接处的物理锯齿。

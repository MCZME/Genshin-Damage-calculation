# 组件库开发规范 (V4.0)

> 本文档为 UI 开发者提供标准组件的开发指南，确保所有新功能模块在视觉与行为上保持高度一致。

---

## 1. 分析磁贴 (Analysis Tiles)

所有分析磁贴必须继承 `core/components/analysis/base_widget.py` 中的 `AnalysisTile` 基类。

### 1.1 生命周期方法
*   `__init__(self)`: 定义磁贴 ID、标题、图标及初始属性（如 `self.expand`）。
*   `load_data(self, adapter)`: 挂载时触发，通过数据适配器拉取所需数据。
*   `update_data(self, *args)`: 外部主动推送新数据时的更新入口。
*   **时间同步**: 不需要显式的 `sync_to_frame`。磁贴应通过 `ft.component` 包装子组件（如游标），并直接读取 `AnalysisState.model.current_frame` 这一 `@ft.observable` 状态来实现自动响应。

### 1.2 包装规范
所有磁贴在使用时必须被 `TileContainer` 包裹。
```python
container = TileContainer(
    tile=your_tile_instance,
    on_close=self._handle_close,
    on_maximize=self._handle_maximize
)
```

## 2. 视觉算法：嵌套收缩 (Nested Inset)

V4.0 引入了模拟金属边框的嵌套算法，开发者在设计新容器时应遵循此逻辑：
1.  **Outer Container**: 设置 `padding=2`，并应用金色线性渐变。
2.  **Inner Container**: 放置实际内容，设置 `clip_behavior=ft.ClipBehavior.ANTI_ALIAS` 并应用 32px 左右的圆角。
3.  **结果**: 内层内容会自动切除外层的 Padding，形成 2px 宽且完美跟随圆角的金属边框。

## 3. 通用控件规范

### 3.1 文本 (Text)
*   **标题**: `size=13, weight=W_800`, 配合 `GenshinTheme.GOLD_DARK`。
*   **正文**: `size=12, weight=W_400`, 配合 `ft.Colors.WHITE` 或 `WHITE70`。
*   **等宽数据**: 显示帧数、时间、伤害数值时，必须使用 `font_family="Consolas"` 防止抖动。

### 3.2 图标 (Icon)
*   优先使用 `_ROUNDED` 后缀的图标（如 `ft.Icons.SETTINGS_ROUNDED`），以匹配柔和的视觉风格。

### 3.3 交互 (Interaction)
*   所有可点击项必须包含悬停（Hover）反馈。
*   **悬停逻辑**: 边框加亮或背景微弱提亮 (`rgba(255,255,255,0.05)`)。

---

## 4. 样式引用准则 (The Rule of Theme)
**严禁**在组件内部硬编码颜色值。必须引用 `ui/theme.py` 中的常量。
*   正例: `color=GenshinTheme.PRIMARY`
*   反例: `color="#D1A2FF"`

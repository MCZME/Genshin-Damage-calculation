# 组件库开发规范 (V5.0)

> 本文档为 UI 开发者提供 MVVM 架构下的组件开发指南，确保代码的高响应性、可维护性与视觉一致性。

---

## 1. 响应式开发模式 (MVVM)

组件不再通过 `AppState` 直接操作底层字典，必须遵循以下链路：

### 1.1 绑定 ViewModel
所有组件应接收一个 `ViewModel` 实例作为 Props。
```python
@ft.component
def YourComponent(vm: YourViewModel):
    # 1. 声明式读取
    return ft.Text(vm.display_name)
```

### 1.2 状态变更通知
在 `ViewModel` 的 Setter 中修改数据后，必须显式调用 `self.notify()` 以触发绑定该 VM 的组件进行 Diff 重绘。

### 1.3 局部重绘优化
- **受控组件**: 涉及滑动（Slider）或高频文本输入（TextField）时，组件内部应使用 `ft.use_state` 维护本地状态，并在 `on_change` 时异步同步给 VM。
- **稳定控件树**: 复杂编辑器必须支持 **Proxy 重新绑定**，禁止通过销毁旧组件来切换数据。

---

## 2. 数据传输规范 (DTO)

对于长列表（Grid/ListView），禁止向列表项传递完整的业务 VM。
- **规范**: 必须使用 `AssetOption` 或类似的轻量级 `dataclass`。
- **原因**: 降低初始化开销，提升滚动性能。

---

## 3. 视觉与交互规范

### 3.1 颜色与图标常量
- **严禁硬编码**: 必须引用 `GenshinTheme` 或 Flet 枚举。
- **命名规范**: Flet 常量统一使用大写加下划线，如 `ft.Colors.BLACK_26`, `ft.Icons.AUTO_AWESOME`。

### 3.2 悬停反馈 (Interaction)
- 所有可点击容器必须提供 `on_hover` 反馈或微小的 `scale` 变化。
- **动画曲线**: 统一使用 `ft.AnimationCurve.DECELERATE` (300ms) 或 `EASE_OUT_BACK` (用于比例缩放)。

---

## 4. 安全性准则
- **线程安全**: 所有异步回调（如持久化保存后更新 UI）必须包裹在 `self.page.run_task(...)` 中。
- **挂载检查**: 手动调用 `.update()` 时必须包裹在 `try...except` 块中，防止 `RuntimeError: Control has no page`。

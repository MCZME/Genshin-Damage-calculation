# 阶段三设计：战果复盘 (Analysis & Review) - V3.2

## 1. 设计概述与哲学
本项目复盘分析页面采用 **“模块化响应式工作台”** 架构，旨在融合 **宏观数据审计 (Macro Dashboard)** 与 **微观时序分析 (Micro Timeline)**。它解决了复杂战斗数据在不同维度下的展示局限性，提供高度可自定义且全场同步的分析体验。

*   **Macro (仪表盘)**: 快速评估队伍强度、伤害构成、循环效率。
*   **Micro (时序轴)**: 精确排查 Buff 覆盖率、元素反应时机、属性面板变化。

## 2. UI 布局架构 (Modern Fluid Layout)

采用 **T 字型现代流式布局**，确保分析的沉浸感与灵活性：

### 2.1 顶层容器 (Header)
*   **全局工具栏**: 包含项目名称、全局 Session 切换。
*   **动态阶段工具 (Phase Tools)**: 仅在复盘模式下显示 **布局预设 (Presets)** 菜单、一键清空画布等功能。

### 2.2 核心工作区 (Workspace)
*   **视图资源库 (View Library)**: 左侧可折叠侧边栏（Mini Drawer）。支持“展开/迷你”两种模式，迷你模式下仅显示图标并提供 Tooltip。
*   **中心画布 (Main Canvas)**: 采用 **12 栅格响应式布局 (Responsive Grid)**。组件根据其业务性格（如曲线图全宽、饼图 1/3 宽）流式排布，空间不足时自动折行。

### 2.3 全局控制层 (Controller)
*   **底部悬浮控制栏**: 采用 **Glassmorphism (玻璃拟态)** 效果，悬浮于画布之上。
*   **功能**: 集成全局时间轴（Slider）、播放/暂停、当前帧数显示（Frame Display）。

## 3. 核心技术实现

### 3.1 响应式栅格系统 (12-Column Grid)
使用 `ft.ResponsiveRow` 实现灵活排版：
*   **全宽视图 (col=12)**: 适用于伤害曲线、动作日志等需要长跨度的组件。
*   **中等视图 (col=6~8)**: 适用于双人对比、大型列表。
*   **窄视图 (col=4)**: 适用于伤害占比饼图、角色属性快照。

### 3.2 帧同步联动机制 (Global Frame Sync)
采用 **“广播 (Broadcasting)”** 机制实现全场数据对齐：
1.  **滑块驱动**: 底部 Slider 广播当前 `frame_id`。
2.  **组件响应**: 每一个 `AnalysisWidget` 接收同步信号，并实时更新内部状态（如同步指示灯变绿、刷新瞬时面板数值）。

### 3.3 布局预设系统 (Layout Presets)
内置多种经典分析模板，一键重排画布：
*   **全视角复盘**: [12/12 曲线] + [6/12 快照] + [6/12 饼图]。
*   **数据审计**: 重点展示伤害构成百分比。

### 3.4 局部参数化配置 (In-widget Configuration)
组件支持“添加后细化”：
*   **配置入口**: 每个 Widget 右上角的设置齿轮。
*   **动态切换**: 支持在组件内部实时切换观测目标（如：从“全队总伤”切换至“指定角色伤害”）。

## 4. 交互规范
1.  **自由添加**: 从资源库点击视图，按推荐占比填入画布。
2.  **全屏最大化**: 点击 Widget 上的“全屏”按钮，隐藏其他组件，当前视图占据 100% 空间。
3.  **联动溯源**: 拖动进度条，全场卡片右上角“绿灯”亮起，同步展示该帧状态。
4.  **布局优化**: 通过 Header 菜单在不同排版模板间快速切换。

## 5. 数据流设计
*   **适配器模式**: `AnalysisView` 持有 `ReviewDataAdapter` 实例并分发给各 Widget。
*   **按需查询**: 
    *   趋势组件初次加载全量曲线。
    *   快照组件在 `sync_frame` 时点对点查询 SQLite 中的特定帧数据。

# 分析界面 (Analysis View) 需求设计说明

> 本文档基于“模块化工作台”理念，定义了仿真引擎 V3.0+ 架构下复盘系统的核心功能与交互规范。

---

## 1. 核心设计哲学 (Design Philosophy)

### 1.1 拒绝死板：按需展示 (On-demand Insight)
*   **痛点**: 每次仿真产生的数据海量且杂乱，固定布局强迫用户处理无关信息。
*   **对策**: 采用“磁贴式仪表盘 (Tile-based Dashboard)”架构。用户根据当前调试目标（如：调充能 vs 调伤害），自行选择开启或关闭特定的分析模块。

### 1.2 深度联动：全局时空同步 (Synchronized Context)
*   **对策**: 引入“全局时间标尺 (Global Scrubber)”。所有激活的模块（图表、投影、审计列表）必须响应同一时间信号，实现“所见即所得”的关联诊断。

---

## 2. 核心交互逻辑 (Core Interaction)

### 2.1 全局时间锚点 (The Global Scrubber) —— 一动全动
*   **定义**: 界面底部（或顶部）存在一个贯穿全仿真时长的控制条。
*   **联动效应**: 拖动标尺时，所有已开启的磁贴（时间轴游标、物理投影实体位置、面板数值快照）必须毫秒级同步响应。
*   **目的**: 提供“电影剪辑级”的逐帧回溯体验，通过单一动作获得全维度的现场反馈。

### 2.2 数据下钻与双向穿演 (Cross-widget Drill-down) —— 点哪到哪
*   **定义**: 组件之间通过“事件 ID”和“帧 ID”建立强逻辑关联。
*   **典型路径**:
    *   **从图表到时间**: 点击 DPS 曲线的波峰 -> 全局锚点自动跳转至对应帧。
    *   **从时间到审计**: 在时间轴点击一个“伤害块” -> 侧边审计面板自动渲染该笔伤害的乘区解构。
    *   **从空间到状态**: 在物理投影区点击一个角色 -> 自动筛选并高亮该角色的能量与 Buff 轨道。
*   **目的**: 消除搜索成本，让数据链路自适应用户的点击意图。

### 2.3 布局灵活性交互 (UX of Tiles) —— 工作台模式
*   **磁贴控制**: 磁贴支持自由排列、关闭、以及双击头部一键最大化。
*   **侧边栏拖拽**: 分析组件以“插件”形式存在于工具箱，用户通过拖拽将其部署至主画布。

## 3. 布局架构方案 (Layout Architecture)

### 3.1 左侧：分析工具箱 (Analysis Toolbox)
*   **设计**: 垂直侧边栏，以图标列形式列出所有可选磁贴。
*   **交互**: 
    *   **开关逻辑**: 点击图标即可在主画布上动态增删对应分析模块。
    *   **折叠支持**: 支持展开/收起，展开时显示磁贴全称，收起时仅保留核心图标以最大化主画布空间。
*   **预设入口**: 提供一键排布预设（如排轴模式、循环模式）的快捷按钮。

### 3.2 底部锚点：时间地平线 (The Scrubber Horizon)
*   **设计**: 全局时间控制标尺横跨屏幕最底部，作为界面的“地平线”。
*   **逻辑**: 无论磁贴布局如何变化，时间控制权永远置顶或处于最易触达层级，确保同步信号的唯一性。

### 3.3 中心：弹性磁贴网格 (Responsive Tile Grid)
*   **12 列栅格协议**: 主画布划分为 12 个等宽垂直列，磁贴按声明的 `col_span` 占据空间：
    *   **Large (12/12)**: 跨越全行（如：多轨时间轴）。
    *   **Medium (6/12)**: 占据半宽（如：物理投影、DPS 曲线）。
    *   **Small (4/12 或 3/12)**: 1/3 或 1/4 宽（如：占比图、全局统计）。
*   **流体堆叠与补位逻辑**:
    *   **左序优先**: 磁贴按添加顺序从左至右、从上至下寻找空位填入。
    *   **溢出换行**: 若当前行剩余列宽不足，新磁贴自动流转至下一行首位。
    *   **自动回填**: 当中间磁贴被关闭时，后续磁贴平滑滑动补位；若后方磁贴尺寸能匹配前方缝隙，则优先上移填充。
*   **一键聚焦 (Focus Mode)**: 双击磁贴头部，磁贴临时最大化至 12/12 宽度并置顶，覆盖其他背景磁贴，再次双击还原原始网格布局。
*   **布局记忆**: 系统自动持久化当前磁贴的排列组合与尺寸配置，实现跨会话的工作台还原。

### 3.4 侧边：弹性详情抽屉 (Dynamic Contextual Drawer)
*   **设计原则**: 空间归还用户。作为 **Overlay (悬浮层)** 存在，彻底脱离主画布布局流，不占据主画布宽度，确保时间轴与物理投影的视觉参考点稳定。
*   **三种交互形态**:
    1.  **收起态 (Collapsed)**: 默认状态。主画布全屏，侧边仅保留微小的交互感应区或垂直图标列。
    2.  **悬浮态 (Floating/On-demand)**: 当用户在磁贴执行“点击下钻”动作时，抽屉从右侧顺滑滑出（占据约 30% 宽度），**覆盖**在主网格之上。
    3.  **固钉态 (Pinned)**: 用户点击抽屉头部的“图钉”图标。在此状态下，点击抽屉外部区域不会触发自动收起，允许用户持续拖动全局标尺并观察详情面板的数值跳变。
*   **逻辑语义**: 作为所有磁贴交互的“共同终点”，承载公式解构、实体状态、事件原始负载等重文本信息。

## 4. 信息密度与视觉层级 (Visual Tiers)

### 4.1 视觉层 (Visual Tier - 极简直观)
*   **适用磁贴**: 物理投影、时间轴泳道。
*   **规范**: 背景极简，通过高对比色块（元素色）表示状态，强调运动感和覆盖关系。

### 4.2 数据层 (Data Tier - 高密结构)
*   **适用磁贴**: 公式审计、事件日志。
*   **规范**: 采用紧凑的表格式排列，利用色块深浅区分乘区或来源，减少视觉扫视成本。

## 5. 技术实现方案 (Technical Implementation)

### 5.1 磁贴标准化协议 (AnalysisTile Protocol)
为了支撑模块化的按需加载，所有分析磁贴必须继承统一基类并实现以下核心接口：
*   `sync_to_frame(frame_id: int)`: 全局标尺变动时的同步回调。
*   `load_data(adapter: ReviewDataAdapter)`: 挂载时的数据预取方法。
*   `get_optimal_size()`: 返回磁贴在网格中的推荐 Slot 尺寸（如 2x1, 1x1）。
*   `on_config_toggle()`: 磁贴内部设置项的交互入口。

### 5.2 高性能同步管线 (The Scrubbing Pipeline)
针对长轴仿真（18000+ 帧）的实时拖动，采取以下优化：
*   **内存索引化**: 磁贴在初始化时将数据库记录转化为“帧-数据”映射表，同步时仅执行 O(1) 复杂度的查找。
*   **非破坏性属性修改**: 严禁在同步时重建控件，仅修改现有控件的视觉属性（如 Position, Color, Value）。
*   **渲染节流 (Throttling)**: 对 `current_frame` 的变化监听执行频率限制，确保在高频拖动下 UI 依然丝滑。

### 5.3 响应式网格引擎 (Layout Engine)
*   **实现方式**: 结合 Flet 的 `ResponsiveRow` 与 `AnimatedSwitcher`。
*   **槽位管理**: 建立 12 列栅格系统。磁贴通过 `col` 属性控制宽度，通过 `visible` 属性控制按需加载。
*   **容器封装**: `TileContainer` 提供统一的头部工具栏（全屏、设置、关闭）以及骨架屏加载态。

### 5.4 跨模块信号总线 (Event Signal System)
*   **中转机制**: 使用 `UIEventBus` 传递下钻意图。
*   **信号流向**: 
    *   [Source Tile] 发出 `ANALYSIS_DRILL_DOWN` (携带 FrameID/EventID)。
    *   [Layout Manager] 拦截信号，驱动全局标尺位移。
    *   [Context Drawer] 接收信号，执行数据查询并滑出展示。

## 6. 动画与交互细节 (Interaction & Motion)

### 6.1 磁贴重组动效 (Tile Orchestration)
*   **入场动画**: 磁贴从工具箱图标位置以 `Scale(0.8->1.0)` 结合 `Fade` 渐变“飞入”槽位，伴随 `EASE_OUT_BACK` 弹性回弹。
*   **流体布局**: 当关闭或移动磁贴引起网格变化时，相邻磁贴通过平滑的位移（而非瞬间跳变）补位，保持视觉注意力的连续性。

### 6.2 全时空同步控制 (Scrubber Ergonomics)
*   **双模控制**: 
    *   **拖拽模式**: 底部标尺支持阻尼感惯性滑动，用于快速定位大跨度时间点。
    *   **微调模式**: 鼠标悬停于标尺或任意磁贴时，通过**滚轮/触控板横滑**实现“逐帧步进”（±1 Frame），满足对 ICD 和附着瞬间的精细审计需求。
*   **关键帧吸附**: 标尺 handle 具备轻微的“磁吸”逻辑，自动感应并吸附至最近的伤害峰值点。

### 6.3 纯悬浮详情容器 (Floating Context Drawer)
*   **物理层级**: 详情抽屉采用 **Overlay (悬浮层)** 模式，彻底脱离主画布布局流。
*   **无感唤起**: 点击下钻点时，抽屉从右侧滑入，覆盖在主画布右侧。**主画布网格不做任何尺寸挤压**，避免触发繁重的响应式布局重绘。
*   **视觉规范**: 
    *   采用强毛玻璃背景（Blur 30+），使下方数据隐约可见但确保文字清晰。
    *   通过深度阴影（Shadow Elevation）建立明确的垂直高度差。
*   **固钉开关**: 即使在悬浮模式下，依然提供“固钉”图标。固钉后点击其他区域抽屉不自动收起，方便用户拖动标尺对比详情。

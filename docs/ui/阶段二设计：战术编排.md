# 阶段二设计：战术编排 (Tactics Phase)

## 1. 阶段概述
阶段二的核心任务是定义实体的行为意图序列。本阶段采用“流式布局”以处理大规模动作序列，并将右栏定义为动作的深度参数编辑器。UI 保持纯净的采集职能，不预测位移，不校验逻辑。

## 2. 布局设计 (Workbench Layout)

### 2.1 左栏：动作库 (Action Library)
*   **核心功能**: 
    *   根据当前队伍成员，列出所有可用的原始动作指令。
    *   **动作模板 (Snippets)**: 支持快速插入预设的连招组合。

### 2.2 中栏：动作序列流 (The Action Flow) —— **核心区域**
*   **设计形态**: **流式瓷砖 (Flow Tiles)**。
    *   动作以小型化卡片形式展示，采用类似文本换行的布局模式，从左至右、自上而下排列。
    *   *优势*: 极大提升单屏动作承载量，适合长达 60s+ 的复杂循环编排。
*   **视觉元素**: 
    *   [角色图标] + [动作名称]。
    *   底色根据执行角色的元素属性动态显示。
*   **交互规范**:
    *   **自由排序**: 选定块后可直接拖拽至序列中任意位置。
    *   **选中联动**: 点击特定动作块，右栏立即载入该块的详细参数。

### 2.3 右栏：动作参数编辑器 (Action Inspector)
*   **功能**: 取代物理画布，专注于单个动作的深度定制。
*   **编辑项**:
    *   **子类型定义**: 如点按/长按切换、多段攻击的具体段落选择。
    *   **静态位移输入**: 若动作包含位移，用户手动录入预期的位移向量或目标点坐标。
    *   **条件备注**: 用于用户标记该动作的意图。

## 3. 核心交互流程
1. 用户在左栏点击动作，动作以“瓷砖块”形式添加到中栏末尾。
2. 用户在中栏通过拖拽优化整体执行顺序。
3. 用户点击中栏中的某个动作块，在右栏进行参数微调。
4. 编排完成后，点击右下角“开始仿真”。

## 4. 设计逻辑：纯净采集
*   **去物理化**: 阶段二不展示任何 CombatSpace 的位置预测。
*   **去逻辑化**: 允许排布任意顺序的动作，后果（如技能放不出、能量不足）统一在阶段三展示。

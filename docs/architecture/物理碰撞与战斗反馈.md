# 架构设计：物理碰撞与战斗反馈

## 1. 2D 几何碰撞判定
所有攻击的有效性判定在 (X, Z) 平面上完成。
- **圆 (Circle)**：基于点到点距离。
- **矩形 (Box)**：基于旋转后的 AABB 或 OBB 判定。
- **扇形 (Sector)**：基于角度差与半径限制。

## 2. 自驱动位移 (Self-Displacement)
- **触发时机**：在技能逻辑执行时。
- **逻辑流**：
    1. 计算技能产生的位移向量。
    2. 更新实体的场景坐标。
    3. 在新位置发起伤害广播。

## 3. 简化物理反馈
- **打击反馈**：仅作为“打击成功”的信号，不产生物理位置移动。
- **停滞处理 (Hitlag)**：可选实现。若实现，则通过暂停实体的 `update()` 计数来模拟。
- **打击类型判定**：如钝击（Blunt）检测，用于触发特定的数值反应（碎冰）。

---
*注：本架构不包含实体间的物理碰撞体积挤压。*

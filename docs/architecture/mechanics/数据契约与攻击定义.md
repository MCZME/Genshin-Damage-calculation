# 架构设计：数据契约与攻击定义 (V2.5)

## 1. 核心契约：AttackConfig (物理与机制契约)
`AttackConfig` 定义了一次攻击的物理本质与机制属性。V2.5 架构下，该契约是连接“原生录入数据”与“仿真计算引擎”的纽带。

### 1.1 字段规范
| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| **`attack_tag`** | `str` | 原始物理标签（如 "普通攻击1"）。用于识别攻击意图。 |
| **`extra_attack_tags`** | `list[str]` | 辅助标签（如 "视为元素爆发伤害"）。用于处理复杂的增伤判定。 |
| **`icd_tag` / `icd_group`** | `str` | 元素附着规则与共享冷却组标识。 |
| **`strike_type`** | `StrikeType` | 打击感类型，决定与盾牌、岩造物的物理交互逻辑。 |
| **`is_ranged`** | `bool` | 是否为远程攻击。 |
| **`hitbox`** | `HitboxConfig` | 攻击的空间碰撞定义。 |

### 1.2 原生映射逻辑 (Data Mapping)
系统在组装阶段执行自动映射，允许开发者直接使用原生术语录入：

| 录入术语 | 内部枚举 (`Enum`) | 物理含义 |
| :--- | :--- | :--- |
| **球** | `AOEShape.SPHERE` | 3D 球体判定 |
| **圆柱** | `AOEShape.CYLINDER` | XZ轴圆判定 + Y轴高度 |
| **长方体** | `AOEShape.BOX` | XYZ 三轴定向判定 |
| **默认 / 突刺 / 切割 / 钝击** | `StrikeType` | 物理交互性质 |

## 2. 行为载体：Damage (伤害契约)
`Damage` 对象封装了一次伤害事件的所有输入参数，但不持有计算逻辑（计算由 `DamagePipeline` 驱动）。

### 2.1 核心属性
- **`element`**: 包含元素类型（`Element`）与元素量（`Element_U`）。
- **`damage_multiplier`**: 基础倍率向量（支持多段或多属性点积）。
- **`scaling_stat`**: 对应的缩放属性名称列表。
- **`data`**: 运行时动态上下文（存储审计链 `audit_trail` 和特殊计算因子）。

### 2.2 多属性点积缩放 (Multi-Stat Scaling)
V2.5 支持复合属性加成（如：攻击 + 精通），在 `Damage` 对象中表现为向量对齐：
```python
# 示例：艾尔海森攻击+精通缩放
Damage(
    damage_multiplier=(120.0, 350.0), # 120%攻击 + 350%精通
    scaling_stat=("攻击力", "元素精通")
)
```

## 3. 物理范围定义：HitboxConfig
描述攻击在 `CombatSpace` 中的覆盖特征。

```python
@dataclass
class HitboxConfig:
    shape: AOEShape = AOEShape.SINGLE
    radius: float = 0.0    # 半径 (SPHERE/CYLINDER)
    height: float = 0.0    # 高度 (CYLINDER/BOX)
    width: float = 0.0     # 宽度 (BOX)
    length: float = 0.0    # 长度 (BOX)
    offset: tuple = (0,0,0)# 相对攻击者原点的位移
```

## 4. 标签解析与判定逻辑
在 `DamagePipeline` 执行期间，系统通过 **`AttackTagResolver`** 对 `AttackConfig` 中的标签进行语义判定：

1.  **语义归约**：将 "普通攻击1"、"普通攻击2" 等非标标签统一归约为 "普通攻击"。
2.  **谓词判定**：通过 `check(target, main, extra)` 方法精准决定当前伤害是否享受特定增伤（如：爆发增伤判定）。
3.  **公式路由**：通过 `is_transformative` 判定当前攻击是否走剧变反应独立公式。

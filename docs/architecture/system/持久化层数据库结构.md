# 持久化层深度优化方案 (Persistence Layer Optimization - V3.0 Architecture)

## 1. 架构核心思想
本项目持久化层采用 **“垂直拆分 (Vertical Partitioning)”** 与 **“强类型隔离 (Strongly Typed Isolation)”** 的设计模式。通过将仿真数据分流为多个动态轨道，解决了大 JSON 快照导致的 I/O 性能瓶颈，并为高精度的伤害审计与动作回溯提供结构化支持。

---

## 2. 会话管理层 (Session & Configuration)

### 2.1 `simulation_sessions` (会话索引表)
存储仿真的全局指标，用于列表展示与快速统计。
*   `id`: INTEGER (PK, AUTOINCREMENT) - 唯一会话标识。
*   `config_name`: TEXT - 仿真配置名称。
*   `created_at`: DATETIME - 运行时间。
*   `total_damage`: REAL - 最终结算总伤害。
*   `duration_frames`: INTEGER - 总持续帧数。
*   `avg_dps`: REAL - 全场平均 DPS。
*   `peak_dps`: REAL - 瞬时最高秒伤。

### 2.2 `simulation_configs` (配置详情表)
专门存储重型 JSON 数据，实现冷热分离。
*   `session_id`: INTEGER (PK, FK) - 关联会话。
*   `config_snapshot`: TEXT - 进入仿真时的全量环境配置快照 (JSON)。

---

## 3. 静态登记层 (Static Entity Registry)

### 3.1 `simulation_entity_registry` (通用注册表)
所有战斗实体的物理身份定义。
*   `session_id`: INTEGER (FK) - 关联会话。
*   `entity_id`: INTEGER - 仿真内唯一实体 ID。
*   `entity_type`: TEXT - 类型 (CHARACTER, TARGET, CONSTRUCT)。
*   `name`: TEXT - 实体名称。
*   `spawn_x`, `spawn_y`, `spawn_z`: REAL - 出生/初始物理坐标。
*   `hitbox_radius`: REAL - 碰撞圆柱底部半径。
*   `hitbox_height`: REAL - 碰撞圆柱总高度。

### 3.2 `simulation_entities` (基础/召唤物特化表)
*   `session_id`, `entity_id`: (PK, FK) - 关联注册表。
*   `owner_id`: INTEGER - 所属者实体 ID (用于伤害溯源)。
*   `created_frame`: INTEGER - 产生帧数。
*   `duration`: INTEGER - 预期存续时长 (帧)。

### 3.3 `simulation_targets` (战斗目标特化表)
*   `session_id`, `entity_id`: (PK, FK) - 关联注册表。
*   `level`: INTEGER - 目标等级。
*   `base_defense`: REAL - 初始防御力基础值。
*   `res_phys`, `res_fire`, `res_water`, `res_wind`, `res_elec`, `res_grass`, `res_ice`, `res_rock`: REAL - 八大平铺抗性列。

### 3.4 `simulation_characters` (角色详细面板表)
*   `session_id`, `entity_id`: (PK, FK) - 关联注册表。
*   `level`: INTEGER - 角色等级。
*   `constellation`: INTEGER - 命座层级 (0-6)。
*   `base_attributes`: TEXT (JSON) - 初始 `attribute_data` 全量面板快照。
*   `weapon_data`: TEXT (JSON) - 武器名称、精炼度、静态被动等。
*   `artifact_sets`: TEXT (JSON) - 激活的套装名称列表。
*   `skill_levels`: TEXT (JSON) - 天赋 A/E/Q 的初始等级。

## 4. 动态数据层 (Dynamic Tracks)

### 4.1 `character_pulses` (角色轨迹轨道)
专门追踪角色的实时空间演进。
*   `session_id`, `frame_id`, `entity_id`: (PK) - 复合主键。
*   `x`, `y`, `z`: REAL - 当前物理位置。
*   `action_id`: TEXT - 当前执行的指令标识 (如 `elemental_skill`)。
*   `is_on_field`: INTEGER - 是否为当前操控角色 (0/1)。

### 4.2 `simulation_state_jumps` (资源跳变轨道)
记录资源（能、血）的非线性变化。
*   `id`: INTEGER (PK, AUTOINCREMENT)。
*   `session_id`: INTEGER (FK)。
*   `frame_id`: INTEGER - 变更时刻。
*   `entity_id`: INTEGER - 关联实体。
*   `jump_type`: TEXT - 类型 (ENERGY, HP)。
*   `new_value`: REAL - 变更后的绝对数值。
*   `delta_value`: REAL - 数值增量。

### 4.3 `modifier_lifecycles` (属性修饰符轨道)
记录属性加成的生效区间，支持离线面板还原。
*   `session_id`, `modifier_id`: (PK) - 唯一 ID 由仿真引擎分配。
*   `entity_id`: INTEGER - 关联实体。
*   `start_frame`, `end_frame`: INTEGER - 生效与失效时刻。
*   `source_name`: TEXT - 加成来源。
*   `stat_type`: TEXT - 修改的属性 (如 `攻击力%`)。
*   `value`: REAL - 修改数值。
*   `op_type`: TEXT - 运算法则 (ADD, MULT, SET)。

### 4.4 效果与护盾层 (Effect & Shield Tracks)

#### **`simulation_effect_lifecycles` (效果生命周期)**
*   `session_id`, `instance_id`: (PK) - 实例唯一 ID 由引擎分配。
*   `entity_id`: INTEGER - 挂载目标。
*   `effect_type`: TEXT - 类型 (SHIELD, STATUS)。
*   `name`: TEXT - 效果名称。
*   `start_frame`, `end_frame`: INTEGER - 存续区间。

#### **`simulation_shield_jumps` (护盾数值跳变)**
*   `session_id`, `instance_id`, `frame_id`: (PK) - 复合主键。
*   `current_shield_hp`: REAL - 实时剩余吸收量。

#### **`target_aura_pulses` (目标附着脉搏)**
*   `session_id`, `frame_id`, `entity_id`: (PK)。
*   `aura_state`: TEXT (JSON) - 目标的元素附着实时状态快照。

## 5. 离散事件与审计层 (Event & Audit Track)

### 5.1 `simulation_event_log` (骨干事件表)
*   `event_id`: INTEGER (PK, AUTOINCREMENT) - 数据库生成。
*   `session_id`, `frame_id`: INTEGER - 触发时刻。
*   `event_type`: TEXT - 类型 (DAMAGE, REACTION, etc.)。
*   `source_id`: INTEGER - 行为发起者。

### 5.2 `event_payloads` (事件负载表)
*   `event_id`: INTEGER (PK, FK)。
*   `payload_json`: TEXT - 原始事件全量 JSON 备份 (冷数据)。

### 5.3 `event_audit_trail` (计算审计明细表)
*   `id`: INTEGER (PK, AUTOINCREMENT)。
*   `event_id`: INTEGER (FK) - 关联具体事件。
*   `modifier_id`: INTEGER - 关联生命周期轨道的具体 ID (系统计算则为 NULL)。
*   `source_name`, `stat_type`: TEXT - 计算因子的名称与属性。
*   `value`: REAL - 涉及的数值。
*   `op_type`: TEXT - 计算法则 (SET, ADD, MULT)。

### 5.4 `event_damage_data` (伤害特化统计表)
*   `event_id`: INTEGER (PK, FK)。
*   `target_id`: INTEGER - 命中目标 ID。
*   `final_damage`: REAL - 最终伤害值。
*   `element_type`: TEXT - 元素类型。
*   `attack_tag`: TEXT - 攻击标签 (NORMAL/SKILL/BURST 等)。
*   `is_crit`: INTEGER - 是否暴击 (0/1)。
*   `reaction_name`: TEXT - 触发反应名称。

#### **`simulation_mechanism_metrics` (表结构)**
| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| `id` | INTEGER (PK) | 自增 ID |
| `session_id` | INTEGER | 关联会话 |
| `frame_id` | INTEGER | 记录时刻 |
| `entity_id` | INTEGER | 关联实体 |
| **`metric_key`** | **TEXT** | **指标名称 (如: "气氛值", "剧团层数")** |
| **`value`** | **REAL** | **当前数值** |

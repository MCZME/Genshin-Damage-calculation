# 伤害审计系统架构 (Damage Audit System Architecture) V2.5.5

## 1. 概述
伤害审计系统是仿真引擎 V2.5 的核心组件，旨在实现从攻击意图到最终数值结算的全流程“可追溯性”。V2.5.5 引入了 **“脱水存储 (Dehydrated Storage)”** 机制，在保证 100% 还原度的前提下，显著降低了 L2 详情表的存储开销。

## 2. 核心设计理念

### 2.1 计算与审计的分离
系统维护两套并行的向量：
- **`stats` (计算向量)**: 存储实时累加值，供 Pipeline 各阶段执行代数运算。
- **`audit_trail` (审计向量)**: 存储 `ModifierRecord`。只有标记为 `audit=True` 的变更才会进入此向量。

### 2.2 脱水存储策略 [S]/[R]
- **[R] (Retrieved - 检索项)**: 
    - **内容**: 基础攻击力、角色等级、初始抗性等面板静态快照。
    - **逻辑**: 在 Stage 2 (Foundation) 阶段抓取，`audit=False`。
- **[S] (Stored - 物理存储项)**: 
    - **内容**: 技能倍率 (Contract)、外部 Buff 注入 (Evolution)、随机暴击判定。
    - **逻辑**: 凡是具有瞬时性或不可预测性的数据，`audit=True`。

## 3. 核心组件实现

### 3.1 DamageContext (审计状态机)
作为 Pipeline 的数据载体，它通过 `add_modifier(source, stat, value, op, audit=True)` 接口统一管辖数值流向。

### 3.2 AuditProcessor (聚合处理器)
负责将过滤后的 `ModifierRecord` 映射并还原为符合《原神》伤害公式的 **7 大标准审计乘区**。

| 标识 (Key) | 乘区名称 | 审计内容 (存储项 [S]) | UI 缝合内容 (检索项 [R]) |
| :--- | :--- | :--- | :--- |
| **BASE** | **基础属性区** | 班尼特 Q 等动态固定值加成 | 角色+武器基础属性、百分比总和 |
| **MULT** | **倍率加值区** | **[全实录]** 技能倍率、申鹤/云堇加值 | (无) |
| **BONUS** | **增伤区** | 莫娜 Q 等动态伤害加成 | 元素伤害加成、动作类型增伤面板 |
| **CRIT** | **暴击区** | **[全实录]** 暴击判定结果 (暴击乘数) | 暴击率、暴击伤害面板 |
| **REACT** | **反应区** | 精通转化的反应加成、特定反应增益 | 反应基础倍率 (增幅/激化) |
| **DEF** | **防御区** | 减防% (动态部分)、无视防御% | 目标防御力面板 [R]、角色等级 [R] |
| **RES** | **抗性区** | 抗性穿透% | 目标基础抗性 [R]、减抗% [R] |

## 4. UI 端数据缝合逻辑 (Hydration)

由于审计链仅物理存储了 `[S]` 项，UI 在渲染伤害详情面板时必须执行以下“缝合”步骤：

### 4.1 状态检索 (Retrieval)
UI 利用伤害事件携带的 `frame_id` 和 `source_id`/`target_id`，从 SQLite 的 `entity_states` 表中拉取该帧的实体属性快照（包含基础属性、面板百分比、等级、抗性等）。

### 4.2 审计应用 (Injection)
将数据库中读取到的 `audit_trail`（包含 `[S]` 项记录）按顺序作用于上述快照之上。
- **示例**：UI 获取到面板攻击力为 2000，审计链中有一条 `[班尼特-美妙旅程] 固定攻击力 +1000`，UI 负责将其合并显示为最终参与计算的 3000。

### 4.3 公式还原 (Formula Reconstruction)
基于缝合后的完整 stats，UI 终端根据 V2.5 标准聚合公式实时渲染公式树：
$$Final = Core\_DMG \times (1 + Bonus\%) \times Crit \times Reaction \times Def \times Res$$

## 5. 开发规范

1. **计算透明度**: 严禁直接修改 `stats` 字典。
2. **过滤准则**: 任何可以通过 `frame_id` 从快照表中唯一确定的属性，在伤害流水线中都必须标记为 `audit=False`。
3. **点积支持**: 在处理多属性缩放（如攻击+精通混合倍率）时，`AuditProcessor` 必须支持从审计链中匹配对应的属性名。
